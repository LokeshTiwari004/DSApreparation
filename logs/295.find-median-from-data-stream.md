# Find Median from Data Stream

- **Problem**: #295 Find Median from Data Stream (Hard)
- **Link**: https://leetcode.com/problems/find-median-from-data-stream/
- **Difficulty**: Hard

## User's Approach

Two-heap approach: `small` (max-heap for lower half) and `large` (min-heap for upper half). Balance so the heaps differ by at most 1. FindMedian returns the average of tops if equal size, or the top of the larger heap.

## Correctness

**Critical bug**: `heapq.heappush_max` and `heapq.heappop_max` **do not exist** in Python's standard library. The code would raise `AttributeError` at runtime. Python's `heapq` only supports min-heaps natively.

The algorithm logic itself (if implemented with correct max-heap semantics) would work, but the balancing scheme is also non-standard — it allows both `small > large` and `large > small` by 1, which complicates the invariants.

The standard fix: push `-num` to `small` to simulate a max-heap, and use `heapq.heappush` / `heapq.heappop` throughout.

## Pattern Analysis

**Label: Two Heaps** — correct for intent. This is the canonical pattern.

The Two Heaps pattern applies when you need to maintain a running median, percentile, or other order statistics from a stream.

## Canonical Solution

```python
class MedianFinder:
    def __init__(self):
        self.small = []   # max-heap (store negatives)
        self.large = []   # min-heap

    def addNum(self, num: int) -> None:
        heapq.heappush(self.small, -num)
        # Move the largest of small to large
        heapq.heappush(self.large, -heapq.heappop(self.small))
        # Rebalance: large should never exceed small in size
        if len(self.large) > len(self.small):
            heapq.heappush(self.small, -heapq.heappop(self.large))

    def findMedian(self) -> float:
        if len(self.small) > len(self.large):
            return -self.small[0]
        return (-self.small[0] + self.large[0]) / 2
```

The canonical approach is simpler and more maintainable:
1. Always push to small (as negative)
2. Pop the max (most negative → most positive after negation) and push to large
3. If large is bigger, pop min from large and push to small
4. Invariant: `len(small) >= len(large)` and `len(small) - len(large) <= 1`

## Key Takeaways

- **Two Heaps** is the canonical pattern for streaming median
- **Max-heap in Python** = push negative values into a min-heap
- `heapq` has no `heappush_max` — the only way is negative values
- The balancing invariant determines which heap's top to return
- This pattern extends to sliding window median (#480), find kth largest (#703)
