# Problem: Edit Distance (#72)

**Difficulty:** Hard  
**Link:** https://leetcode.com/problems/edit-distance/  
**Date Solved:** 07/07/26  
**Phase:** Phase 3 – Hard Track

---

## User's Approach

Top-down DP with memoization (recursive):
- `dp(i, j)` = min edits to convert `word1[i:]` to `word2[j:]`
- Base cases: if one string exhausted, remaining edits = length of the other
- If characters match: no operation needed, `dp(i+1, j+1)`
- If mismatch: `1 + min(insert=dp(i, j+1), delete=dp(i+1, j), replace=dp(i+1, j+1))`
- Store results in dict keyed by `(i, j)`

---

## Correctness

✅ **Correct** — handles all cases:
- Empty strings → 0
- One empty → length of the other
- Identical strings → 0
- All mismatch conversions

---

## Pattern Analysis

**Pattern:** DP (Edit Distance) — **Correct Label**

**Why DP applies:**
- Optimal substructure: edit dist of suffixes depends on edit dist of smaller suffixes
- Three operations (insert/delete/replace) map to three subproblems
- Overlapping subproblems: same `(i, j)` pairs computed multiple times without memoization

## Complexity

| Metric | User's Claim | Actual | Optimal |
|--------|--------------|--------|---------|
| Time   | O(mn)        | O(mn)  | O(mn)   |
| Space  | O(mn)        | O(mn)  | O(min(m,n)) |

✅ **Time is optimal.** Space can be optimized to O(min(m,n)) with bottom-up 2-row DP.

## Code Quality

**Strengths:**
- Correct recurrence and base cases
- Clean recursive structure
- Memoization via dict is straightforward

**Issues:**
1. **Redundant lookup** — line 24 `dp()` result is unused, then `lookup[(0,0)]` re-accesses it. Just `return dp()`
2. **Dict vs lru_cache** — `@lru_cache` would be more Pythonic than manual dict for this pattern
3. **Top-down recursion limit** — for very long strings (500+500), recursion depth could approach 1000, hitting Python's default limit
4. **Bottom-up is more standard** for edit distance in interviews

**Canonical Implementation (Bottom-up 2-row DP):**
```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m, n = len(word1), len(word2)
        if m < n:
            word1, word2, m, n = word2, word1, n, m
        
        prev = list(range(n + 1))
        for i in range(1, m + 1):
            curr = [i] * (n + 1)
            for j in range(1, n + 1):
                if word1[i-1] == word2[j-1]:
                    curr[j] = prev[j-1]
                else:
                    curr[j] = 1 + min(prev[j], curr[j-1], prev[j-1])
            prev = curr
        return prev[n]
```

## Key Takeaways

1. **Edit Distance recurrence** — `dp(i,j) = match ? dp(i+1,j+1) : 1 + min(insert, delete, replace)`
2. **Three operations** — insert (same i, j+1), delete (i+1, j), replace (i+1, j+1)
3. **Bottom-up 2-row** — O(min(m,n)) space is the canonical interview answer
4. **Top-down vs bottom-up** — both work; bottom-up avoids recursion depth limits
5. **Canonical Hard DP** — appears in many forms (regular expression matching, wildcard matching)
