# Problem: Container With Most Water (#11)

**Difficulty:** Medium  
**Link:** https://leetcode.com/problems/container-with-most-water/  
**Date Solved:** 01/07/26  
**Phase:** Phase 3 – Mixed Set 1

---

## User's Approach

Two-pointer from opposite ends, always moving the shorter side inward:
1. `lo, hi` at both ends, track `res`
2. At each step, compute area = `min(height[lo], height[hi]) * (hi - lo)`
3. If left height < right height: move `lo` right (the shorter side)
4. Else: move `hi` left (the shorter side)
5. Return max area

---

## Correctness

✅ **Correct** — handles all cases:
- Monotonically increasing heights → moves hi left
- Monotonically decreasing heights → moves lo right
- Equal heights → can move either (both produce same result)
- Single-element array not possible (2 ≤ n ≤ 10⁵ per constraints)

---

## Pattern Analysis

**Pattern:** Two Pointers — **Correct Label**

**Why it applies:**
- The container area is bounded by the shorter wall and the width
- Moving a pointer inward reduces width, so the only way to increase area is to find a taller wall
- By always moving the shorter wall, we never miss a potential larger area

## Key Takeaways

1. **Two-pointer from edges** — always move the shorter side inward
2. **Area = min(L, R) * width** — width decreases each step, so only height improvements matter
3. **Greedy choice works** — the shorter side is the bottleneck; moving it can find a taller wall
4. **O(n) O(1)** — optimal and minimal
