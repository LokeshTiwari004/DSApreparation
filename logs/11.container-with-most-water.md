# 11. Container With Most Water

## Problem
- **Title:** Container With Most Water
- **Number:** #11
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/container-with-most-water/

## User's Approach
Two pointers starting at opposite ends. At each step, computes water using the shorter height × current width. Moves the pointer pointing to the shorter height inward. Tracks the maximum water seen.

## Correctness
Correct. The greedy argument holds: moving the taller pointer inward can only decrease width without any guarantee of increasing height, so it can never produce a larger container. Moving the shorter pointer is the only way to potentially find a taller boundary. Handles single element (loop exits immediately with `lo == hi`, returns 0).

## Pattern Analysis
**Correct pattern: Two Pointer (converging).**
Classic converging two-pointer. The key insight is that the bottleneck is always the shorter side — moving it is the only viable exploration strategy. Width strictly decreases each step, so the greedy choice is provably optimal.

## Complexity
- **Time:** O(n) — pointers move inward until they meet
- **Space:** O(1) — two index variables
- User stated O(n) / O(1) — **correct**

## Code Quality
Clean and readable. Variables `hright`, `hleft` are clear. `while lo <= hi` is correct (at `lo == hi`, width = 0, water = 0, contributes nothing). No issues.

## Key Takeaways
- Converging two pointers: start at extremes, move the weaker/smaller side inward
- The greedy proof: moving the taller side inward cannot improve the result (shorter side remains the bottleneck, width only decreases)
- Width always decreases → the only gain is from height → move the shorter pointer
- This pattern applies to any "optimize over a range that shrinks" type problem
