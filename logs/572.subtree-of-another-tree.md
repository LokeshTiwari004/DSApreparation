# 572. Subtree of Another Tree

## Problem
- **Title:** Subtree of Another Tree
- **Number:** #572
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/subtree-of-another-tree/

## User's Approach
Reuses `isSameTree` from #100 (same XOR null-check pattern). `isSubtree` checks if current root matches subRoot, OR recurses on right, OR recurses on left. If `root is None`, returns `False`.

## Correctness
Correct. The OR short-circuit means once a match is found, further recursion stops. The base case `root is None` handles the case where we've exhausted the tree without finding a match.

## Pattern Analysis
**Pattern: Tree Traversal + Structural Comparison.**
Combines two patterns:
1. **Tree traversal** — walk every node of root (pre-order)
2. **Structural comparison** — `isSameTree` checks if two subtrees are identical

This is the standard approach. Time complexity is O(m × n) because `isSameTree` can be called at every node of root. The optimal O(m + n) approach uses serialization (KMP on preorder with markers) but is more complex.

## Complexity
- **Time:** O(m × n) — at each of m nodes in root, isSameTree can compare up to n nodes of subRoot
- **Space:** O(m) — recursion stack; worst-case m for skewed tree
- Optimal: O(m + n) with serialization + KMP string matching
