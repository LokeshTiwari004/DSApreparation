# Problem: Valid Palindrome II (#680)

**Difficulty:** Easy  
**Link:** https://leetcode.com/problems/valid-palindrome-ii/  
**Date Solved:** 21/06/26  
**Phase:** Phase 3 – Mixed Set 1

---

## User's Approach

Two-pointer from opposite ends with a checkpoint system:
1. When a mismatch is found, save `(lo, hi)` as `ckpt` and try skipping the left char (`lo += 1`)
2. On second mismatch, restore from `ckpt` and try skipping the right char (`hi -= 1`) instead — set `flag = True`
3. On third mismatch, return `False`
4. If pointers cross without issues, return `True`

---

## Correctness

✅ **Correct** — handles all cases:
- Already palindrome → returns True
- Single deletion needed (left skip OR right skip works) → returns True
- Both skips fail → returns False
- Empty/single char → satisfied by `while lo < hi` (no loop iterations, returns True)

---

## Pattern Analysis

**Pattern:** Two Pointers (Greedy) — **Correct**

**Why it applies:**
- Palindrome checking is the canonical two-pointer case (opposite ends, compare inward)
- The "allow one deletion" twist means you check both possible deletions when a mismatch occurs

**Canonical pattern:** Two-pointer with at-most-one-skip palindrome check

## Code Quality

**Strengths:**
- O(1) space with no extra data structures
- Early exit on failure

**Issues:**
1. **State machine is non-standard** — `ckpt`/`flag`/`else` branching makes the control flow harder to follow than the canonical solution
2. **Tricky logic on second mismatch** — the `elif ckpt:` / `else:` branches are easy to misread. First mismatch enters `else` (left skip). Second mismatch enters `elif` (right skip from checkpoint). This relies on side-effect ordering.
3. **No helper function** — `is_palindrome(s[lo+1:hi+1])` or equivalent substring check is the standard clean approach

**Canonical Implementation:**
```python
class Solution:
    def validPalindrome(self, s: str) -> bool:
        lo, hi = 0, len(s) - 1
        while lo < hi:
            if s[lo] != s[hi]:
                return self._is_pal(s, lo + 1, hi) or self._is_pal(s, lo, hi - 1)
            lo += 1
            hi -= 1
        return True

    def _is_pal(self, s: str, lo: int, hi: int) -> bool:
        while lo < hi:
            if s[lo] != s[hi]:
                return False
            lo += 1
            hi -= 1
        return True
```

---

## Key Takeaways

1. **Two-pointer palindrome** is the simplest O(n) O(1) pattern — always expand inward
2. **At-most-one-skip** = try both `(lo+1, hi)` and `(lo, hi-1)` when mismatch found
3. **Separate helper function** makes the "skip or not" logic self-documenting vs inline state machine
4. The state machine approach works but is harder to verify correct at a glance — prioritize readability in interviews
5. O(n) time, O(1) space — can't do better for this problem
