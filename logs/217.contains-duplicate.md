# 217. Contains Duplicate

## Problem
- **Title:** Contains Duplicate
- **Number:** #217
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/contains-duplicate/

## User's Approach
Single pass using a HashSet. For each number, checks if it's already in the set. If yes, returns True immediately (early exit). Otherwise adds it. Returns False if no duplicate found.

## Correctness
Correct. Early exit on first duplicate is optimal. Handles empty arrays (returns False), single element (returns False), all duplicates (returns True on second element).

## Pattern Analysis
**Correct pattern: HashSet for membership tracking.**
Classic "have I seen this before?" pattern. A set gives O(1) average lookup and insertion. Early exit makes it optimal in the best case.

## Complexity
- **Time:** O(n) — at most one full pass
- **Space:** O(n) — set stores up to n elements
- User stated O(n) time / **O(1) space — WRONG**
- The set grows linearly with input. O(1) space means no extra data structure that grows with n. Fix this in your mental model.

## Code Quality
Clean. The only issue is the incorrect space complexity note. No code issues.

## Canonical Solution
User's solution is canonical. The only alternative worth knowing:
```python
# One-liner (less readable but occasionally seen)
def containsDuplicate(nums):
    return len(nums) != len(set(nums))
```
This is O(n) time/space but no early exit — worse in practice for large inputs with early duplicates.

## Key Takeaways
- O(1) space means the extra memory used does NOT grow with input size (e.g., a few integer variables)
- A set that stores elements is always O(n) space in the worst case
- Early exit (`return True` on first hit) is better than building full set and checking length
- HashSet is the canonical structure for "seen before?" queries
