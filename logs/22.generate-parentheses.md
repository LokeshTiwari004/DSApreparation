# 22. Generate Parentheses

## Problem
- **Title:** Generate Parentheses
- **Number:** #22
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/generate-parentheses/

## User's Approach
Backtracking with two tracked variables: `n` (remaining '(' to place) and `open` (currently unclosed '(' count). At each step, either add '(' if `n > 0`, or add ')' if `open > 0`. Base case: `open == 0 and n == 0`.

## Correctness
Correct and canonical. The two invariants are:
1. You can only add '(' if you haven't used all n opening brackets
2. You can only add ')' if there's an unmatched '(' (open > 0)
3. When both are 0, the string is balanced and complete

## Pattern Analysis
**Pattern: Backtracking with constraint tracking.**
The state is `(n, open)` — remaining opens and current balance. Canonical version:
```python
def generateParenthesis(self, n):
    result = []
    def backtrack(path, left, right):
        if len(path) == 2 * n:
            result.append(''.join(path))
            return
        if left < n:
            path.append('(')
            backtrack(path, left + 1, right)
            path.pop()
        if right < left:
            path.append(')')
            backtrack(path, left, right + 1)
            path.pop()
    backtrack([], 0, 0)
    return result
```
The standard approach tracks `left` (placed '(') and `right` (placed ')') and avoids `open` as a derived state. Both are equivalent.

## Complexity
- **Time:** O(4^n / sqrt(n)) — nth Catalan number, ≈ O(4^n / n^(3/2))
- **Space:** O(n) — recursion stack + path string
- User left complexity blank

## Code Quality
Very clean (5 min solve). The state parameters `(n, open)` are intuitive. String concatenation `path+')'` creates new strings at each level (O(n²) total) — using a list with `append`/`pop` + `''.join` is more efficient but this is fine for n ≤ 8.

## Key Takeaways
- Generate Parentheses = Catalan number problem: count = C(2n, n)/(n+1)
- Two constraints: can't close before opening, can't exceed n opens
- The `open` counter tracks balance — must stay ≥ 0
- Same pattern as #78 but with two counters instead of index
- 5 min solve — the recursive pattern is fully internalized
