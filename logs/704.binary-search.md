# 704. Binary Search

## Problem
- **Title:** Binary Search
- **Number:** #704
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/binary-search/

## User's Approach
`while True` loop with `lower`, `upper`, `cursor`. Moves `lower = cursor` or `upper = cursor` (without the ±1 offset). When `upper - lower < 2` (adjacent pointers), manually checks both slots and returns -1 if neither matches.

## Correctness
Correct — passes all tests. The solution converges because the cursor always lands strictly between lower and upper, so the gap shrinks by at least 1 each iteration until the adjacent-case fires. However the logic is more complex than necessary due to the non-standard bound updates.

## Pattern Analysis
**Correct pattern: Binary Search.**
Standard binary search halves the search space each iteration. The key invariant: target, if present, is always within `[lo, hi]`. Moving `lo = mid + 1` or `hi = mid - 1` *excludes* mid from the next search (mid was already checked and wasn't the target), which keeps the invariant and naturally terminates when `lo > hi`.

User's variant uses `lo = mid` and `hi = mid` (inclusive), so mid is never excluded. This requires the special adjacent-case block to avoid an infinite loop at gap = 1.

## Complexity
- **Time:** O(log n) — search space halves each iteration
- **Space:** O(1) — three index variables
- User stated **O(n log n) — wrong**. This is a significant error. O(n log n) is the complexity of sorting algorithms like merge sort. Binary search makes O(log n) comparisons, each O(1). Never confuse the two.

## Code Quality
Works but the `while True` + special adjacent block adds ~8 lines to a 7-line problem. The complexity note error is the most important thing to fix. Rewrite using the standard template:

```python
def search(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = lo + (hi - lo) // 2   # avoids integer overflow vs (lo+hi)//2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            lo = mid + 1             # exclude mid, search right half
        else:
            hi = mid - 1             # exclude mid, search left half
    return -1
```
`lo + (hi - lo) // 2` vs `(lo + hi) // 2`: both work in Python (no integer overflow), but the former is the industry-standard habit from C/Java — use it.

## Key Takeaways
- Binary search is **O(log n)** — never O(n log n). Conflating these two is a red flag in interviews
- Standard template: `lo = mid + 1`, `hi = mid - 1` — excludes mid, no special cases needed
- Loop condition `while lo <= hi` naturally terminates: when lo > hi, search space is empty
- `mid = lo + (hi - lo) // 2` is the safe midpoint calculation — build the habit now
- This exact template is the base for all binary search variants (#35, #153, #33, #875)
