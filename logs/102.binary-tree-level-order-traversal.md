# 102. Binary Tree Level Order Traversal

## Problem
- **Title:** Binary Tree Level Order Traversal
- **Number:** #102
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/binary-tree-level-order-traversal/

## User's Approach
BFS with deque storing `(node, level)` tuples. Uses a dict keyed by level to collect values. After BFS, reconstructs the list of lists by iterating levels in order.

## Correctness
Correct. BFS guarantees level-order processing. Dict + reconstruction produces correct output.

## Pattern Analysis
**Pattern: BFS with level tracking.**
Standard approach avoids level tuples by processing one level at a time:
```python
def levelOrder(self, root):
    if not root: return []
    result = []
    q = deque([root])
    while q:
        level = []
        for _ in range(len(q)):
            node = q.popleft()
            level.append(node.val)
            if node.left: q.append(node.left)
            if node.right: q.append(node.right)
        result.append(level)
    return result
```
`for _ in range(len(q))` captures the level count before children are added, naturally separating levels without tuple tracking.

## Complexity
- **Time:** O(n) — each node visited once
- **Space:** O(n) — queue + result storage

## Key Takeaways
- BFS level-order: `for _ in range(len(q))` separates levels naturally
- No need for level numbers when queue provides implicit ordering
- Dict-based level collection is a Python trick but less efficient than building result during BFS
- This is the canonical "queue per level" tree pattern
