# 994. Rotting Oranges

## Problem
- **Title:** Rotting Oranges
- **Number:** #994
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/rotting-oranges/

## User's Approach
BFS using two sets (`visiting` and `will_visit`) swapped at each level. Starts from all initial rotten oranges. At each level, rots adjacent fresh oranges. Counts minutes via level count. Returns -1 if any fresh oranges remain.

## Correctness
Correct. BFS guarantees minimum minutes since all oranges rot simultaneously per level. Edge cases handled: no fresh oranges (returns 0), unreachable fresh oranges (returns -1).

## Pattern Analysis
**Pattern: Multi-source BFS (level-order).**
Standard deque-based approach:
```python
def orangesRotting(self, grid):
    m, n = len(grid), len(grid[0])
    q = deque()
    fresh = 0
    for r in range(m):
        for c in range(n):
            if grid[r][c] == 2:
                q.append((r, c))
            elif grid[r][c] == 1:
                fresh += 1
    if fresh == 0:
        return 0
    minutes = 0
    dirs = [(1,0), (-1,0), (0,1), (0,-1)]
    while q:
        minutes += 1
        for _ in range(len(q)):
            r, c = q.popleft()
            for dr, dc in dirs:
                nr, nc = r + dr, c + dc
                if 0 <= nr < m and 0 <= nc < n and grid[nr][nc] == 1:
                    grid[nr][nc] = 2
                    fresh -= 1
                    q.append((nr, nc))
    return -1 if fresh else minutes - 1
```
The deque version is more conventional: `for _ in range(len(q))` processes levels, `dirs` array replaces 4 if-checks.

## Complexity
- **Time:** O(m×n) — each cell visited at most once
- **Space:** O(m×n) — worst-case if all cells are rotten (queue size)

## Code Quality
Correct but non-standard BFS implementation:
- Set-based instead of deque — unusual but functional
- `minutes = -1` init is confusing — standard uses `0` with post-level adjustment
- Four separate if-checks instead of a dirs array — verbose
- Correct early exit for `fresh_count == 0`
- 30 min, no hints — over 20-min rule

## Key Takeaways
- Multi-source BFS = start BFS from all rotten oranges simultaneously
- Level tracking = minute counting (each BFS level = 1 minute)
- Direction array (`dirs`) simplifies 4-directional grid traversal
- `while q` + `for _ in range(len(q))` is the standard level-processing pattern
- Returns -1 if any fresh remains → BFS doesn't guarantee full coverage
