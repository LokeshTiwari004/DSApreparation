# 643. Maximum Average Subarray I

## Problem
- **Title:** Maximum Average Subarray I
- **Number:** #643
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/maximum-average-subarray-i/

## User's Approach
Fixed-size sliding window in C. Computes initial window sum for the first `k` elements, then slides by adding the incoming element and subtracting the outgoing element (`sum += nums[cursor] - nums[cursor - k]`). Tracks maximum sum throughout.

## Correctness
Correct. Edge case check `numsSize < k` is good defensive coding (problem guarantees it won't happen, but good habit). Returns `(double)max / k` with correct cast to avoid integer division.

## Pattern Analysis
**Correct pattern: Fixed-Size Sliding Window.**
Textbook fixed window: enter one element, exit one element per step. Avoids recomputing the full sum each time (would be O(n × k)). The incremental update is the key optimization.

## Complexity
- **Time:** O(n) — one pass after O(k) initialization
- **Space:** O(1) — no extra data structures
- User stated O(n) / O(1) — **correct**

## Code Quality
Correct but **written in C**. Should be Python for interview prep. The C cast `(double)max / k` is necessary in C to avoid integer division — in Python this is handled automatically with `/`. Rewrite:

```python
def findMaxAverage(nums, k):
    window_sum = sum(nums[:k])
    max_sum = window_sum
    for i in range(k, len(nums)):
        window_sum += nums[i] - nums[i - k]
        max_sum = max(max_sum, window_sum)
    return max_sum / k
```

## Key Takeaways
- Fixed sliding window: O(n) instead of O(n × k) brute force — always use incremental update
- Formula: `new_sum = old_sum + incoming - outgoing`
- In Python, `/` always returns float — no cast needed
- This is the simplest fixed-window problem; internalize it as the template for all fixed-window problems
