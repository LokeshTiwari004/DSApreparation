# 3. Longest Substring Without Repeating Characters

## Problem
- **Title:** Longest Substring Without Repeating Characters
- **Number:** #3
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/longest-substring-without-repeating-characters/

## User's Approach
Sliding window with a hashmap storing the last seen index of each character. When a duplicate is found inside the current window (`seen[char] >= start`), moves `start` to one past the duplicate's last position. Tracks max window length at each step.

## Correctness
Correct. The guard `seen[char] > start - 1` (equivalent to `seen[char] >= start`) correctly handles the case where a duplicate was seen before the current window — without this check, `start` could move backwards. Handles empty string (loop doesn't execute, `ans` stays 0). All edge cases handled.

## Pattern Analysis
**Correct pattern: Sliding Window with HashMap.**
The window expands right with each character. When a duplicate is found *inside* the window, the left boundary jumps to exclude the previous occurrence. The hashmap enables O(1) lookup of the last seen position — more efficient than a set-based approach that would need to shrink one element at a time.

## Complexity
- **Time:** O(n) — each character visited at most twice (once added, once via start jump)
- **Space:** O(min(n, charset)) — dict stores at most charset-size entries (26 for lowercase, 128 for ASCII)
- User stated O(n) time — **correct**
- User stated O(1) space — **wrong**. The dict grows with the character set. O(1) space means fixed memory regardless of input. Correct notation: O(charset) or O(min(n, charset)).

## Code Quality
Clean and idiomatic. The `seen[char] > start - 1` guard is smart and correct. One minor style note: `seen[char] >= start` is slightly more readable and conventional.

## Key Takeaways
- Sliding window with hashmap: O(n) vs O(n²) brute force; the map gives O(1) jump for left pointer
- The guard `seen[char] >= start` is critical — without it, start can regress (bug)
- Space complexity involving a dict/set is O(size of stored data), not O(1)
- "charset" as space complexity is acceptable in interviews for string problems
