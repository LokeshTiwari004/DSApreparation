# 199. Binary Tree Right Side View

## Problem
- **Title:** Binary Tree Right Side View
- **Number:** #199
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/binary-tree-right-side-view/

## User's Approach
BFS with two deques (`current_lvl`, `next_lvl`). Process left-to-right within a level. When `current_lvl` is exhausted, the last node popped is the rightmost — append its value. Swap queues and continue.

## Correctness
Correct. The last node popped from each level is guaranteed to be the rightmost since left child is enqueued before right child.

## Pattern Analysis
**Pattern: Tree BFS (Level Order Traversal).** BFS naturally visits nodes level by level. Capturing the last node of each level gives the right-side view.

## Complexity
- **Time:** O(n) — each node visited once
- **Space:** O(n) — at most one level of nodes in queue at a time

## Code Quality
Correct but non-standard. Using two deques with queue swapping works but the canonical approach uses a single deque and a level-size loop:

```python
for _ in range(len(q)):
    node = q.popleft()
    if node.left: q.append(node.left)
    if node.right: q.append(node.right)
answer.append(node.val)  # after loop, node is rightmost
```

This pattern is cleaner and was already seen in #102 Binary Tree Level Order Traversal. Missing `List` and `Optional` type imports from `typing` (handled by LeetCode platform but worth noting).

## Key Takeaways
- Right side view = last node at each BFS level
- Single-queue approach with level-size tracking is the standard BFS pattern
- Left-to-right processing means the last node in a level is the rightmost
