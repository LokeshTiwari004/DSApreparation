# 75. Sort Colors

## Problem
- **Title:** Sort Colors
- **Number:** #75
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/sort-colors/

## User's Approach
Custom three-pointer scheme using `B` (pointer to earliest 1) and `C` (pointer to earliest 2), with `idx` as cursor. On encountering 0: cascades swaps involving both B and C pointers. On encountering 1: swaps with C pointer and repositions B. Tracks pointer initialization state with `None`.

## Correctness
Passes all tests. However, the solution is extremely hard to verify for correctness by reasoning alone — the multiple `None` guards and cascaded swap logic make it brittle. In an interview, reviewers would struggle to follow this and it raises red flags about maintainability.

## Pattern Analysis
**Correct pattern: Dutch National Flag (DNF) Algorithm — Dijkstra.**
DNF is a classic three-pointer partitioning algorithm: `lo` (next slot for 0), `mid` (cursor), `hi` (next slot for 2). Everything left of `lo` is 0; everything right of `hi` is 2; between `lo` and `mid` is 1. The loop invariant makes correctness trivially provable.

This is a named algorithm that interviewers will specifically ask about. Reinventing it with a novel scheme loses points even if it works.

## Complexity
- **Time:** O(n) — single pass (`mid` never revisits elements)
- **Space:** O(1) — in-place, three index variables
- User stated O(n) / O(1) — **correct**

## Code Quality
30+ lines for a 6-line problem. Multiple `None` checks, cascaded conditionals, hard to trace. In an interview setting this would likely not be accepted as clean even if correct.

## Canonical Solution
```python
def sortColors(nums):
    lo, mid, hi = 0, 0, len(nums) - 1
    while mid <= hi:
        if nums[mid] == 0:
            nums[lo], nums[mid] = nums[mid], nums[lo]
            lo += 1
            mid += 1
        elif nums[mid] == 1:
            mid += 1
        else:  # nums[mid] == 2
            nums[mid], nums[hi] = nums[hi], nums[mid]
            hi -= 1
            # do NOT increment mid: swapped element from hi is unexamined
```
Memorize this. The key insight: when swapping with `hi`, don't advance `mid` because the incoming element is unknown.

## Key Takeaways
- Dutch National Flag is a named algorithm — know it by name and implement it in < 2 minutes
- The invariant: `[0..lo-1]` = 0s, `[lo..mid-1]` = 1s, `[mid..hi]` = unexamined, `[hi+1..n-1]` = 2s
- When swapping with `hi`, do NOT advance `mid` — the swapped-in element hasn't been examined
- When swapping with `lo`, advance both `lo` and `mid` — incoming was a 1 (already examined range)
- Reinventing standard algorithms in interviews signals unfamiliarity with CS fundamentals
