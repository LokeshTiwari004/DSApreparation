# Problem: Sum of Square Numbers (#633)

**Difficulty:** Medium  
**Link:** https://leetcode.com/problems/sum-of-square-numbers/  
**Date Solved:** 22/06/26  
**Phase:** Phase 3 – Mixed Set 1

## User's Approach

Iterate `i` from 0 to `sqrt(c/2) + 1`, check if `c - i*i` is a perfect square:
1. Upper limit `ul = int((c // 2) ** 0.5) + 1`
2. For each `i`, compute `c - i*i` and check if square root is integer via `.is_integer()`
3. Return True if found, else False

---

## Correctness

✅ **Correct** for all valid inputs (c ≥ 0):
- `c = 0` → `ul = 1`, loop runs `i=0`, checks `0 - 0 = 0`, `sqrt(0).is_integer()` = True → returns True
- `c = 1` → `ul = 1`, checks `1 - 0 = 1` → True
- `c = 2` → `ul = 1`, checks `2 - 0 = 2` → False, then `i=1` not reached → False (but 2 = 1² + 1² should be True — **BUG**)

**Bug found:** Upper bound `sqrt(c/2)` is incorrect. For `c=2`, `sqrt(2/2) = 1`, loop only runs `i=0`, misses `i=1`. Correct upper bound is `floor(sqrt(c))`.

Test case `c=2` would fail with this implementation.

---

## Pattern Analysis

**Pattern:** Two Pointers / Math — **Partially Correct Label**

**Why it applies:**
- The canonical solution uses two pointers: `lo = 0`, `hi = floor(sqrt(c))`, move inward based on sum vs target
- Current solution is a single-loop with math check — technically works (if upper bound fixed) but not the standard two-pointer pattern

**Canonical pattern:** Two pointers on range `[0, sqrt(c)]` — move `lo` up when sum < c, `hi` down when sum > c

## Code Quality

**Strengths:**
- Concise (6 lines of logic)
- Uses Python's `.is_integer()` cleanly for perfect square check
- No extra data structures

**Issues:**
1. **Bug in upper bound** — `sqrt(c/2)` misses valid pairs where one square > c/2 (e.g., c=2, 5, 10)
2. **Non-canonical pattern** — single loop with math check instead of two-pointer
3. **Precision risk** — floating point `** 0.5` + `is_integer()` can have edge cases for large numbers; integer arithmetic preferred

**Canonical Implementation (Two Pointers):**
```python
class Solution:
    def judgeSquareSum(self, c: int) -> bool:
        lo, hi = 0, int(c ** 0.5)
        while lo <= hi:
            curr = lo * lo + hi * hi
            if curr == c:
                return True
            elif curr < c:
                lo += 1
            else:
                hi -= 1
        return False
```

---

## Key Takeaways

1. **Two-pointer on squares** — start with `lo=0, hi=floor(sqrt(c))`, move inward like sorted array two-sum
2. **Upper bound is `sqrt(c)`** not `sqrt(c/2)` — one square can be > c/2 (e.g., 2 = 1² + 1²)
3. **Integer arithmetic preferred** — `lo*lo + hi*hi` avoids floating point precision issues
4. **Two-pointer is more canonical** for "sum of two squares" — easier to explain in interviews
5. **O(sqrt(c)) time, O(1) space** — optimal for this problem
