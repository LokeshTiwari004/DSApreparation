# Problem: Binary Tree Inorder Traversal (#94)

## User's Approach

Recursive inorder traversal with list concatenation:
- Base case: empty node → `[]`
- Recursive: `inorder(left) + [root.val] + inorder(right)`

---

## Correctness

✅ **Correct** — produces correct inorder traversal for all tree shapes.

---

## Pattern Analysis

**Pattern:** Trees (DFS, Inorder) — **Correct Label**

**Why it applies:**
- Inorder = left → root → right
- Recursive DFS is the most natural expression

---

## Complexity

| Metric | User's Claim | Actual | Optimal |
|--------|--------------|--------|---------|
| Time   | O(n)         | O(n²) worst-case | O(n) |
| Space  | O(n)         | O(n)   | O(n)    |

⚠️ **Time is O(n²) for skewed trees** — `+` creates a new list copying all elements at each level. Each node's value is copied depth times. A helper with `.append()` avoids this.

---

## Code Quality

**Strengths:**
- Clean, minimal, readable
- Correct base case

**Issues:**
1. **List concatenation O(n²)** — `left + [root.val] + right` copies all elements into a new list at each call. Use a helper that appends to a shared list.
2. **No iterative version** — the iterative stack approach is worth knowing for interviews

**Canonical Implementation (Recursive):**
```python
class Solution:
    def inorderTraversal(self, root: Optional[TreeNode]) -> List[int]:
        res = []
        def dfs(node):
            if not node:
                return
            dfs(node.left)
            res.append(node.val)
            dfs(node.right)
        dfs(root)
        return res
```

**Canonical Implementation (Iterative):**
```python
class Solution:
    def inorderTraversal(self, root: Optional[TreeNode]) -> List[int]:
        res, stack = [], []
        cur = root
        while cur or stack:
            while cur:
                stack.append(cur)
                cur = cur.left
            cur = stack.pop()
            res.append(cur.val)
            cur = cur.right
        return res
```

---

## Key Takeaways

1. **Avoid `+` for list building in recursion** — O(n²) for skewed trees; use `.append()` instead
2. **Helper function pattern** — collect results via closure list, not concatenation
3. **Iterative stack** — `while cur or stack` with nested `while cur` to go left, pop, go right
4. **Three traversal orders** — the iterative pattern adapts: preorder (process at push), inorder (process at pop), postorder (two-stack / reverse)
