# Largest Rectangle in Histogram

- **Problem**: #84 Largest Rectangle in Histogram (Hard)
- **Link**: https://leetcode.com/problems/largest-rectangle-in-histogram/
- **Difficulty**: Hard

## User's Approach

Two-pass monotonic stack:
1. **Left pass**: find the nearest smaller element to the left (NSL) for each bar using an increasing-height stack
2. **Right pass**: find the nearest smaller element to the right (NSR) in reverse
3. **Compute**: width = (right_bound - left_bound - 1), area = height × width, track max

User notes: 30 mins, studied the solution, noted O(4n) time / O(3n) space.

## Correctness

Correct. The monotonic stack correctly identifies the "nearest smaller" boundary on each side. The width formula `right_bound - left_bound - 1` correctly gives the number of bars the current height can span.

Edge cases handled:
- Increasing heights — NSL defaults to -1, NSR defaults to n, width = n for the last element with max height
- Decreasing heights — each bar bounded only by neighbors
- Equal heights — `<` comparison means equal heights don't count as boundaries (correct, since a bar of equal height has the same extension as the neighbor, handled when the neighbor is processed)

Potential subtle issue: The left and right passes are interleaved in the same loop (odd pattern) which makes the code harder to follow than two separate loops.

## Pattern Analysis

**Label: Monotonic Stack** — correct. This is the canonical pattern.

Monotonic stack is used when you need to find the "next greater/smaller element" in O(n) time. Here it finds the nearest smaller element on each side to compute the maximum width a given height can extend.

## Complexity

| Aspect | User's | Optimal |
|--------|--------|---------|
| Time | O(4n) = O(n) | O(n) |
| Space | O(3n) = O(n) | O(n) |

Correct big-O but the constants are off. Each element is pushed and popped at most once per pass (2× push, 2× pop = 4 operations = O(n)). Space is O(n) for two stacks + widths array.

There is also a **single-pass** version that uses one stack and computes area on the fly when a smaller height is encountered. It's more elegant but harder to derive.

## Code Quality

- **Good notes**: time spent (30 min), hint used (studied solution), complexity, pattern documented
- **Interleaved left/right pass** in one loop is clever but reduces readability — two separate loops are clearer
- Variable names `left` and `right` for stacks are opaque — `stack` or `mono_stack` is standard
- The `widths[i] += i - left_bound - 1` and `widths[n-i-1] += right_bound - (n-i-1) - 1` combo is triply indexed — easy to make off-by-one errors
- No dead code or debug prints

## Canonical Solution (Single-Pass)

```python
class Solution:
    def largestRectangleArea(self, heights: List[int]) -> int:
        stack = []  # monotonic increasing stack of indices
        max_area = 0
        heights.append(0)  # sentinel to force final computation

        for i, h in enumerate(heights):
            while stack and heights[stack[-1]] > h:
                height = heights[stack.pop()]
                width = i if not stack else i - stack[-1] - 1
                max_area = max(max_area, height * width)
            stack.append(i)

        heights.pop()
        return max_area
```

Key insight: when a bar is popped because a smaller bar arrives, its right boundary is the current index and its left boundary is the new top of stack. The sentinel `0` at the end ensures all remaining bars in the stack are processed.

## Key Takeaways

- **Monotonic stack** + **NSL/NSR** pattern is the standard approach
- Width = right_boundary - left_boundary - 1 for each bar
- The **single-pass** variant is more elegant — process on pop, use sentinel value
- The `<` (strict) vs `<=` comparison determines boundary semantics for equal heights
- This pattern generalizes to #85 Maximal Rectangle (2D extension)
