# 133. Clone Graph

## Problem
- **Title:** Clone Graph
- **Number:** #133
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/clone-graph/

## User's Approach
DFS traversal storing clones in a dict keyed by `node.val`. For each original node, creates a duplicate, stores it, recursively traverses neighbors, then appends cloned neighbors.

## Correctness
Correct for LeetCode's constraints where node values are unique (1-indexed). Edge case handling for empty and singleton graphs is explicit but partially redundant with the DFS logic.

## Pattern Analysis
**Pattern: Graph DFS with clone map.**
Standard DFS approach uses `node` as dict key and returns clone directly:
```python
def cloneGraph(self, node):
    if not node:
        return None
    clone = {}
    def dfs(node):
        if node in clone:
            return clone[node]
        cp = Node(node.val)
        clone[node] = cp
        for neighbor in node.neighbors:
            cp.neighbors.append(dfs(neighbor))
        return cp
    return dfs(node)
```
Key differences from user's approach:
1. Uses `node` as key (not `node.val`) — works for any node identity
2. DFS returns the clone — eliminates the dict lookup
3. No early-return edge case — DFS handles everything naturally

## Complexity
- **Time:** O(V + E) — each node and edge visited once
- **Space:** O(V) — clone dict + recursion stack
- User's "O(Nodes) time, O(Nodes) space" is approximately correct

## Code Quality
Correct but uses `node.val` as dict key (works for LC but fragile in general). The `traverse` function is side-effect-based (returns None) requiring a separate dict lookup for neighbor appending. The early-return `not node.neighbors` check is unnecessary noise. 30 min, no hints.

## Key Takeaways
- Clone graph = DFS + hashmap of node → clone (prevents infinite cycles)
- Use `node` as dict key, not `node.val` — nodes are unique by identity
- Return clone from DFS directly — no side-effect pattern
- BFS (queue) is also valid for clone graph
- The hashmap serves double duty: visited tracking + clone storage
