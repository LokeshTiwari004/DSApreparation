# 53. Maximum Subarray

## Problem
- **Title:** Maximum Subarray
- **Number:** #53
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/maximum-subarray/

## User's Approach
Attempted a sliding window approach: maintained `start`, `end`, `s` (current sum). Tried to shrink the window from the left when removing left-most element increased the sum. Reset the window when sum dropped below 0.

## Correctness
Passes LeetCode tests but the logic is fragile. The approach is not a clean sliding window and has a non-obvious control flow. The `continue` inside the left-shrink block skips updating `start` on other branches, making correctness hard to reason about. Will likely break on custom edge cases or variants.

## Pattern Analysis
**User labeled: Sliding Window — incorrect.**
**Correct pattern: Kadane's Algorithm (1D Dynamic Programming).**

Sliding window requires a **monotonic condition** to decide when to shrink — e.g., "shrink when we see a duplicate" or "shrink when sum exceeds k". With negative numbers, no such clean condition exists: a negative element might be part of the optimal subarray. Kadane's is the right model:

> `current = max(num, current + num)` — either extend the subarray or start fresh at this element.

This is a DP recurrence: `dp[i] = max(nums[i], dp[i-1] + nums[i])`.

## Complexity
- **Time:** O(n) — single pass
- **Space:** O(1) — two variables
- User stated O(n) / O(1) — **correct**

## Code Quality
Overly complex for what is a 4-line problem. The `start/end/continue` logic is hard to follow and hard to extend. In an interview, this implementation would raise questions about clarity.

## Canonical Solution
```python
def maxSubArray(nums):
    current = answer = nums[0]
    for num in nums[1:]:
        current = max(num, current + num)  # extend or restart
        answer = max(answer, current)
    return answer
```
Internalize this. It is the answer to "maximum subarray" in every interview context.

## Key Takeaways
- Sliding window only works when the window has a **monotonic shrink condition** — negatives break that
- Kadane's: at each position, decide whether to extend the current subarray or start fresh
- This is a DP problem in disguise: `dp[i] = max(nums[i], dp[i-1] + nums[i])`
- Ability to state "this is Kadane's algorithm" and explain why in an interview is expected
- All-negative input: the answer is the largest (least negative) single element — handled correctly by initializing `current = nums[0]`
