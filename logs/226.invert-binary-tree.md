# 226. Invert Binary Tree

## Problem
- **Title:** Invert Binary Tree
- **Number:** #226
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/invert-binary-tree/

## User's Approach
Recursive. Swap left and right children, recurse on each. Base case: null root returns null. Python tuple unpacking enables one-line swap after recursive calls.

## Correctness
Correct. Every node is visited exactly once and its children are swapped. Base case handles empty tree. The recursion naturally terminates when it reaches leaf nodes (children are None, invertTree returns None).

## Pattern Analysis
**Correct pattern: Recursive Binary Tree Traversal (DFS).**
This is the fundamental tree recursion pattern. The three lines — base case, recursive operation, return root — form the template for virtually all recursive tree problems.

## Complexity
- **Time:** O(n) — each node visited once
- **Space:** O(n) — recursion stack depth equals tree height; worst case O(n) for a skewed tree
- User stated O(n) / O(n) — **correct**

## Code Quality
Canonical. The tuple-unpacking swap is elegant: Python evaluates both recursive calls on the right before assigning, so no temporary variable needed. Notes are clean.

## Key Takeaways
- Tree recursion template: base case → recursive call → return root
- Python tuple swap eliminates temp variables for tree inversion
- O(n) space comes from the recursion stack, not explicit data structures
- This is the simplest tree problem — builds foundation for all recursive tree traversal
- "Invert Binary Tree" is famously associated with Google interviews (Max Howell story)
