# Trapping Rain Water

- **Problem**: #42 Trapping Rain Water (Hard)
- **Link**: https://leetcode.com/problems/trapping-rain-water/
- **Difficulty**: Hard

## User's Approach

Pre-compute prefix max (`leftmax`) and suffix max (`rightmax`) arrays, then for each position compute `min(leftmax[i], rightmax[i]) - height[i]` — the classic DP solution.

## Correctness

Correct. Handles all cases:
- Empty array — early return 0
- Ascending/descending — prefix/suffix max correctly gives 0 water at edges
- Plateau — `min(leftmax, rightmax) - height` correctly yields 0 where ground blocks water

## Pattern Analysis

**Label: Two Pointers / Monotonic Stack** — correct in intent, but the implementation is **Prefix/Suffix Max (DP)**, not two pointers.

The canonical pattern labels for this problem are:
1. **Prefix/Suffix Max (DP)** — O(n) time, O(n) space (what was implemented)
2. **Two Pointers** — O(n) time, O(1) space (optimal)
3. **Monotonic Stack** — O(n) time, O(n) space (alternative)

## Complexity

| Aspect | User's | Optimal |
|--------|--------|---------|
| Time | O(n) | O(n) |
| Space | O(n) | O(1) |

Time is optimal but space can be O(1) with the two-pointer variant. Worth learning the two-pointer version.

- Clean, minimal, readable

## Canonical Solution (Two-Pointer, O(1) space)

```python
class Solution:
    def trap(self, height: List[int]) -> int:
        left, right = 0, len(height) - 1
        left_max = right_max = water = 0
        while left < right:
            if height[left] < height[right]:
                if height[left] >= left_max:
                    left_max = height[left]
                else:
                    water += left_max - height[left]
                left += 1
            else:
                if height[right] >= right_max:
                    right_max = height[right]
                else:
                    water += right_max - height[right]
                right -= 1
        return water
```

Key insight: at any position, the water level is bounded by the smaller of the two "max seen from each side". Since we only care about the smaller side's max, we can move inward from the side with the smaller current height.

## Key Takeaways

- **Prefix/Suffix Max** is the most intuitive O(n) solution — compute "max from left" and "max from right" for each position
- **Two Pointer** reduces space to O(1) by tracking max from each side on the fly
- The water at position i depends only on the *min* of the two surrounding max heights — never the larger one
- The two-pointer version is a common follow-up — know both
- This problem also has a monotonic stack variant (decreasing height stack)
