# 150. Evaluate Reverse Polish Notation

## Problem
- **Title:** Evaluate Reverse Polish Notation
- **Number:** #150
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/evaluate-reverse-polish-notation/

## User's Approach
Stack of integers. `match`/`case` on each token. For operators: pop two operands, compute, push result. For numbers: convert to int and push. Division truncation toward zero handled with a floor-adjust hack: `b // a`, then `+1` if negative with remainder.

## Correctness
Correct. The subtraction trick `-stack.pop() + stack.pop()` (equivalent to `a - b`) is clever. The division handling correctly detects negative results with remainder and adds 1 to convert Python's floor division to truncation-toward-zero. Passes all 22 test cases.

## Pattern Analysis
**User labeled: Stack of integers.**
**Correct label: Stack (expression evaluation).**
RPN evaluation is the canonical stack application — operands go on stack, operators consume them. The order of operands matters for subtraction and division: `a - b` requires knowing which popped value is `a` (first pushed, second popped).

## Complexity
- **Time:** O(n) — each token processed once
- **Space:** O(n) — stack holds up to n/2 values
- User stated O(n) / O(n) — **correct**

## Code Quality
Clean use of Python 3.10+ `match`/`case`. The division trick handles truncation correctly but is non-obvious. The simpler alternative is `int(b / a)` — `int()` truncates toward zero in Python:

```python
case '/':
    a = stack.pop()
    b = stack.pop()
    stack.append(int(b / a))
```

One line, no conditional logic, no floor-division edge cases.

## Key Takeaways
- RPN + stack = canonical pairing — numbers go on stack, operators consume and produce
- For non-commutative ops (-, /), order matters: `stack.pop()` gives second operand, careful with the operation
- `int(b / a)` truncates toward zero — simpler than floor-division + correction
- Python `match`/`case` is valid in interviews if the interviewer is OK with modern Python (3.10+)
- Pattern extends to infix evaluation (Shunting Yard algorithm)
