# 141. Linked List Cycle

## Problem
- **Title:** Linked List Cycle
- **Number:** #141
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/linked-list-cycle/

## User's Approach
Fast and slow pointer (Floyd's Cycle Detection). Slow moves 1 step, fast moves 2 steps. If they meet, a cycle exists. If fast or fast.next becomes None, no cycle. Took 1 hint.

## Correctness
Correct. The null check order is right: checks `slow_pt.next` before advancing slow, and `fast_pt.next and fast_pt.next.next` before advancing fast. The early `if not head: return False` handles empty list. One style issue: `id(slow_pt) == id(fast_pt)` compares memory addresses — works in CPython but `slow_pt is fast_pt` is the idiomatic Python identity check.

## Pattern Analysis
**Correct pattern: Fast & Slow Pointers (Floyd's Cycle Detection).**
Floyd's algorithm: if a cycle exists, fast (2x speed) will eventually lap slow (1x speed) and they will meet inside the cycle. If no cycle, fast will reach the end. Time: O(n). The meeting point has significance in extended problems (#142 — find cycle start).

## Complexity
- **Time:** O(n) — fast pointer traverses at most 2× the list length before meeting slow or reaching end
- **Space:** O(1) — two pointer variables
- User stated O(n) / O(1) — **correct**

## Code Quality
Functional but the `while True` loop with internal break conditions is less readable than a standard while condition. The `id()` comparison should be `is`. Cleaner version:

```python
def hasCycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            return True
    return False
```
4 lines, one loop condition, standard Python identity check.

## Key Takeaways
- Fast & slow pointer: the go-to pattern for cycle detection in any sequence
- `slow is fast` not `id(slow) == id(fast)` — use Python's `is` for object identity
- `while fast and fast.next` is the standard null guard for 2-step movement
- Advance fast BEFORE checking equality — if both start at head, they'd falsely match at step 0
- Extended problem: #142 asks for the *node* where cycle begins — requires a second phase after meeting
