# 98. Validate Binary Search Tree

## Problem
- **Title:** Validate Binary Search Tree
- **Number:** #98
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/validate-binary-search-tree/

## User's Approach
Recursive range-constrained DFS. Each node receives a `lo` (exclusive lower bound) and `hi` (exclusive upper bound). Uses `None` as sentinel for unbounded sides.

## Correctness
Correct. Handles all BST constraints. The `<=` and `>=` checks correctly reject duplicates (BST typically requires strict inequality).

## Pattern Analysis
**Pattern: BST Validation (Range Constraint).** The canonical approach uses `-inf`/`+inf` sentinels that narrow as you traverse left/right. An alternative is inorder traversal — a valid BST must yield a strictly increasing sequence.

## Complexity
- **Time:** O(n) — each node visited once
- **Space:** O(n) — recursion stack (worst-case skewed tree)

## Code Quality
Logic is correct but the recursive calls on line 28 are overcomplicated:
```python
# User's version:
check(node.left, lo, min(node.val, hi) if hi else node.val)
# Standard:
check(node.left, lo, node.val)
```
The bound propagation is simpler than the user made it — left child's upper bound is always `node.val`, no min/ternary needed. Pattern label "MIN-Max Game" is non-standard — use "BST Range Constraint" or "Inorder Traversal."

## Key Takeaways
- BST validation requires checking range, not just child-parent comparison
- Range propagates naturally: left → upper = node.val, right → lower = node.val
- Both recursive range-check and inorder traversal work; inorder is simpler for follow-ups
