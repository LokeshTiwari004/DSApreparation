# 125. Valid Palindrome

## Problem
- **Title:** Valid Palindrome
- **Number:** #125
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/valid-palindrome/

## User's Approach
Two-pointer from opposite ends. Skip non-alphanumeric characters using `isalnum()`. Compare lowercase versions of characters. Return False on mismatch, True if pointers cross.

## Correctness
Correct. Handles empty strings, strings with only non-alphanumeric characters, mixed case, and punctuation.

## Pattern Analysis
**Pattern: Two Pointers (opposite ends).** Standard palindrome checking pattern. The `isalnum()` + `lower()` calls handle the problem's constraint to ignore non-alphanumeric chars and case.

## Complexity
- **Time:** O(n) — single pass
- **Space:** O(1) — no extra data structures

## Code Quality
Clean and canonical. Standard two-pointer setup, clear skip logic with `continue`. Variable names `lo`/`hi` are standard. Could merge the skip logic into the while condition (`while lo < hi and not s[lo].isalnum(): lo += 1`) but current approach is equally readable.

## Key Takeaways
- Palindrome with filtering is still a two-pointer problem
- `isalnum()` checks both letters and digits — covers the "alphanumeric" requirement
- Two-pointer from edges is optimal for palindrome checking
