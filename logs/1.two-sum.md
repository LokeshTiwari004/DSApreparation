# 1. Two Sum

## Problem
- **Title:** Two Sum
- **Number:** #1
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/two-sum/

## User's Approach
Single-pass hashmap. For each number, computes the complement (`target - num`) and checks if it already exists in the map. If yes, returns the pair of indices. Builds the map on the fly.

## Correctness
Correct. Handles all cases including negative numbers and duplicate values (as long as indices differ, which the problem guarantees). Return value `return ip, i` yields a tuple — LeetCode accepts it, but the expected return type is `List[int]`. Use `return [ip, i]`.

## Pattern Analysis
**Correct pattern: HashMap / Complement Lookup.**
The problem reduces to: for each element, does its complement exist among elements seen so far? A hashmap gives O(1) lookup per element. The single-pass variant (check before insert) avoids using the same element twice.

## Complexity
- **Time:** O(n) — one pass, O(1) HashMap operations
- **Space:** O(n) — map stores up to n entries
- User stated O(n) / O(n) — **correct**

## Code Quality
Clean and idiomatic. `hm.get(target-num)` with `is not None` check is correct (handles the case where index 0 is stored). Minor: return a list not a tuple.

## Canonical Solution
User's solution is essentially canonical. Reference for clarity:
```python
def twoSum(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
```

## Key Takeaways
- HashMap complement lookup is the go-to pattern for "find pair with property X"
- Single-pass: check before insert to avoid using the same index twice
- `dict.get()` vs `in` — both work; `get` is useful when you need the value directly
- Always return the correct type (`List[int]`, not tuple)
