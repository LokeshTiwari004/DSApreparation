# 100. Same Tree

## Problem
- **Title:** Same Tree
- **Number:** #100
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/same-tree/

## User's Approach
Recursive comparison using XOR for null-check branching. Return `False` if exactly one node is `None` (XOR), `True` if both are `None`, and recursively compare values + left and right subtrees otherwise.

## Correctness
Correct. Covers all cases:
- Both `None` → `True` (empty trees)
- One `None` → `False`
- Both non-`None` → compare value and recurse on subtrees

The `elif` after XOR is technically redundant (XOR being `False` means either both `None` or both not `None`), but it doesn't cause bugs and reads clearly.

## Pattern Analysis
**Pattern: Recursive Tree Traversal (structural comparison).**
Canonical pattern uses simpler null logic:
```python
def isSameTree(self, p, q):
    if not p and not q:
        return True
    if not p or not q:
        return False
    return p.val == q.val and self.isSameTree(p.left, q.left) and self.isSameTree(p.right, q.right)
```
The `if not p or not q` catches the "exactly one is None" case since both-None was already returned. The XOR variant is equivalent but less common — both are correct.

## Complexity
- **Time:** O(n) — each node visited once across both trees
- **Space:** O(n) — recursion stack up to tree height; worst-case O(n) for skewed tree
- User correctly noted O(n) space

## Code Quality
Clean at 5 lines of logic. The XOR `^` on booleans works (Python treats `True ^ False` as `True`) but is non-standard. 20 min, no hints — within the 20-min rule. Notes include base case and edge case documentation.

## Key Takeaways
- Tree comparison recursion template: base cases (both None / one None) → recurse on children
- The XOR trick is neat but `if not p or not q` is more readable and expected in interviews
- O(n) space from recursion stack is correct — don't claim O(1)
- This pattern (recursive structural comparison) extends directly to #572 Subtree of Another Tree
