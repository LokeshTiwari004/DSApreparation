## User's Approach

Binary search on answer space `[1, max(piles)]`:
1. `lo, hi = 1, max(piles)`
2. `while lo < hi`: compute `mid`, simulate hours with ceiling division
3. Feasibility: `sum((p + mid - 1) // mid for p in piles) <= h`
4. If `hrs > h`: `lo = mid + 1` (too slow)
5. Else: `hi = mid` (fast enough, try slower)

---

## Correctness

✅ **Correct** — handles all cases:
- h = len(piles) → speed = max(piles)
- h large → speed = 1
- All edge cases covered

## Pattern Analysis

**Pattern:** Binary Search on Answer — **Correct Label**

**Why it applies:**
- Monotonic: faster speed always decreases or maintains hours
- Need minimum feasible speed
