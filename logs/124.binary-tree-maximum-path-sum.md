# Binary Tree Maximum Path Sum

- **Problem**: #124 Binary Tree Maximum Path Sum (Hard)
- **Link**: https://leetcode.com/problems/binary-tree-maximum-path-sum/
- **Difficulty**: Hard

## User's Approach

Post-order DFS: at each node, compute the best path through the node as a "connector" (left + node + right), and return the best "branch" (node + max(left, right, 0)). A global variable tracks the overall maximum seen.

## Correctness

The core logic is correct — post-order traversal explodes all possible paths, and the "branch vs connector" distinction ensures no path is double-counted or invalid.

Edge cases handled correctly:
- Single node tree — returns node.val
- All negative values — returns the least negative node
- Negative child values — automatically excluded by `max(left, right, 0)` in the return value (branch path)
- Positive-only trees — sums correctly propagate

However, the guard clauses `left_max if node.left else self.max_path_sum` inside the `max()` call are a workaround — if a child is absent, the code avoids passing 0 into the comparison. This is unnecessary with the standard pattern (`max(dfs(node.left), 0)`) which filters 0 at the call site.

## Pattern Analysis

**Label: Trees / DFS** — correct.

This is a **post-order DFS** problem where each node returns the "best single-branch path" upward, while a side effect updates the global maximum with the "connector path" (both branches + node). This recursive pattern is canonical for tree path problems.

## Code Quality

Issues:

1. **Non-standard initialization**: `self.max_path_sum = root.val` works but `float('-inf')` is safer and idiomatic
2. **Redundant guard clauses**: `left_max if node.left else self.max_path_sum` is convoluted — the standard `max(dfs(node.left), 0)` pattern obviates this entirely
3. **Redundant outer max**: Line 26 `self.max_path_sum = max(DFS(root), self.max_path_sum)` does nothing — the global max was already updated during the DFS call
4. **No notes** — no comments or time/hint metadata in the file

## Canonical Solution

```python
class Solution:
    def maxPathSum(self, root: Optional[TreeNode]) -> int:
        self.max = float('-inf')

        def dfs(node):
            if not node:
                return 0
            left = max(dfs(node.left), 0)   # ignore negative branches
            right = max(dfs(node.right), 0)
            self.max = max(self.max, node.val + left + right)  # connector path
            return node.val + max(left, right)  # best branch to parent

        dfs(root)
        return self.max
```

Key differences from user's solution:
- `float('-inf')` init — safe for any input
- `max(dfs(node.left), 0)` at call site eliminates per-node guards
- Single `max()` comparison on the connector path
- Clean `not node` base case instead of conditional calls
- No redundant operations

## Key Takeaways

- **Post-order DFS** is the pattern for "best path" problems in trees — children report up, parent assembles
- **Two values from each node**: the "connector" (both branches + node) updates the global max, the "branch" (node + best single branch) returns upward
- **Clamp negative contributions to 0** — a path never needs to include a negative sum subpath
- **Global variable** for the result is idiomatic in DFS tree problems (avoids tuple returns)
- The canonical 7-line version is cleaner than the 12-line version — remove redundancy
