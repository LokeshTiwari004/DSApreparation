# 242. Valid Anagram

## Problem
- **Title:** Valid Anagram
- **Number:** #242
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/valid-anagram/

## User's Approach
Build a frequency map from `s`. Then decrement counts while iterating `t` — return False if a character in `t` isn't in the map. Final pass checks all counts are 0.

## Correctness
Correct. Handles mismatched lengths implicitly (extra characters in `t` not found in map → False; extra characters in `s` → non-zero count → False). Early length check `if len(s) != len(t): return False` would be a fast path optimization but is not required for correctness.

## Pattern Analysis
**Correct pattern: HashMap / Frequency Map.**
Anagram check = same character frequencies. Build frequency map for one string, subtract for the other, verify all zeroes. Conceptually identical to "are these multisets equal?"

## Complexity
- **Time:** O(n + m) — one pass over each string
- **Space:** O(charset) — map bounded by alphabet size (26 for lowercase)
- User stated O(n) / O(charset) — **correct**

## Code Quality
Clean and correct. Minor: `ref[char] += 1` / `else: ref[char] = 1` is more clearly written using `collections.Counter` or `dict.get`:
```python
ref[char] = ref.get(char, 0) + 1
```
Three separate loops (build, subtract, verify) works but can be reduced to two with an early length check. No meaningful issues.

## Canonical Solution
```python
from collections import Counter

def isAnagram(s, t):
    return Counter(s) == Counter(t)
```
One line. Use this in interviews unless asked to implement from scratch.

## Key Takeaways
- Frequency map is the canonical structure for "same characters, same counts?" problems
- `Counter` from `collections` is the Pythonic tool — know it
- Early length check saves work: if `len(s) != len(t)`, immediately return False
- This pattern is the building block for #49 Group Anagrams and #438 Find All Anagrams
