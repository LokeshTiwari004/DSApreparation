# 36. Valid Sudoku

## Problem
- **Title:** Valid Sudoku
- **Number:** #36
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/valid-sudoku/

## User's Approach
Three dicts of sets keyed by digit (1–9): `x` tracks rows per digit, `y` tracks columns, `z` tracks 3×3 blocks. Iterate all 81 cells, check membership, add on first encounter.

## Correctness
Correct. Handles all Sudoku constraints (row, column, 3×3 block) with early exit.

## Pattern Analysis
**Pattern: Hashing / Set-based validation.** Uses hash sets to track seen positions per digit. The board is fixed-size (9×9), so any O(1)-space approach is trivially O(1).

## Complexity
- **Time:** O(1) — fixed 81 cells
- **Space:** O(1) — 9 digits × 3 dicts, each with at most 9 entries

## Code Quality
Correct but variable naming is opaque: `x`, `y`, `z` give no semantic information. Use `rows`, `cols`, `blocks` for readability. The dict-of-sets keyed by digit value is an unusual but valid structure — most solutions use a single set of tuples `(row, val)`, `(col, val)`, `(block, val)` for cleaner code. No pattern label in header notes.

## Key Takeaways
- Sudoku validation is a set-membership problem across 3 constraints
- The block index `(i//3, j//3)` maps a cell to its 3×3 sub-box
- For fixed-size boards, O(1) complexity analysis is correct but the same approach generalizes to O(n²)
