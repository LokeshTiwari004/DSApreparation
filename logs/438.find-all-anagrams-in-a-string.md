# 438. Find All Anagrams in a String

## Problem
- **Title:** Find All Anagrams in a String
- **Number:** #438
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/find-all-anagrams-in-a-string/

## User's Approach
Sliding window with two frequency dictionaries. Build pattern dict for `p`, initialize window dict with first `plen` chars of `s`. Slide: remove left char, decrement/del from window dict, add right char, compare dicts.

## Correctness
Correct. The sliding window maintains exactly `plen` characters at each position. Dict comparison correctly detects anagram matches.

## Pattern Analysis
**Pattern: Sliding Window + Frequency Map.**
Standard approach with `collections.Counter`:
```python
def findAnagrams(self, s, p):
    p_count = Counter(p)
    s_count = Counter()
    result = []
    for i in range(len(s)):
        s_count[s[i]] += 1
        if i >= len(p):
            if s_count[s[i-len(p)]] == 1:
                del s_count[s[i-len(p)]]
            else:
                s_count[s[i-len(p)]] -= 1
        if s_count == p_count:
            result.append(i - len(p) + 1)
    return result
```
`Counter` provides cleaner dict management. The fixed-size window pattern is: add right → remove left → compare.

## Complexity
- **Time:** O(n) — each character processed once; dict comparison is O(charset) but charset ≤ 26 for lowercase
- **Space:** O(charset) — two fixed-size frequency dicts

## Code Quality
Correct but some style issues:
- `chr` shadows the Python built-in `chr()` function
- Manual dict increment with `if chr in dict: ... else: ...` — `dict.get(chr, 0) + 1` or `Counter` is cleaner
- Unnecessary `continue` statements in increment blocks

## Key Takeaways
- Sliding window + freq map = standard for substring anagram detection
- Fixed-size window: add right char, remove left char, compare
- `collections.Counter` simplifies dict management
- `chr` is a Python built-in — don't shadow it
