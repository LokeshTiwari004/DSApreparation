## Pattern Analysis
**Correct pattern: Fixed-Size Sliding Window + Frequency Map.**
This is a direct extension of #242 (Valid Anagram):
- Instead of checking one pair of strings, check every window of `len(p)` in `s`
- Maintain a frequency map of `p` and a sliding frequency map of the current window
- When the two maps match, the window start index is an anagram

The optimized approach avoids recomputing the full map on every slide — just add the incoming character and remove the outgoing character.

```python
from collections import Counter

def findAnagrams(s, p):
    if len(p) > len(s):
        return []
    
    p_count = Counter(p)
    window = Counter(s[:len(p)])
    result = []
    
    if window == p_count:
        result.append(0)
    
    for i in range(len(p), len(s)):
        # add incoming
        window[s[i]] += 1
        # remove outgoing
        outgoing = s[i - len(p)]
        window[outgoing] -= 1
        if window[outgoing] == 0:
            del window[outgoing]
        if window == p_count:
            result.append(i - len(p) + 1)
    
    return result
```

## Complexity
- **Time:** O(n) — each character enters and exits the window once; map comparison is O(charset) = O(26) = O(1) for lowercase
- **Space:** O(charset) — two frequency maps bounded by alphabet size

## Key Takeaways
- Fixed sliding window = window size doesn't change, only slide by 1 each step
- Reuse #242's anagram check logic but apply it to every window position
- Avoid full map rebuild on every slide — incremental add/remove is O(1) per step
- Deleting zero-count keys keeps map comparison clean
- This problem bridges #242 (Valid Anagram) and #76 (Minimum Window Substring)
