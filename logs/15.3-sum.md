# 15. 3Sum

## Problem
- **Title:** 3Sum
- **Number:** #15
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/3sum/

## User's Approach
Hash map mapping value → set of indices. Double loop over all pairs (i, j), compute complement `-a-b`, check if complement exists at a different index. Dedup via sorted tuple stored in a set. Special case for triple-zero.

## Correctness
Correct. The index tracking via hash map ensures each triplet uses three distinct positions. The set-of-tuples dedup works. The (0,0,0) special case is necessary for triple-zero.

## Pattern Analysis
**Pattern: Hash Map + Pair Enumeration.**
Standard approach uses sorting + two pointers — no hash map needed:
```python
def threeSum(self, nums):
    nums.sort()
    result = []
    for i in range(len(nums) - 2):
        if i > 0 and nums[i] == nums[i-1]:
            continue
        left, right = i + 1, len(nums) - 1
        while left < right:
            s = nums[i] + nums[left] + nums[right]
            if s < 0: left += 1
            elif s > 0: right -= 1
            else:
                result.append([nums[i], nums[left], nums[right]])
                while left < right and nums[left] == nums[left+1]: left += 1
                while left < right and nums[right] == nums[right-1]: right -= 1
                left += 1; right -= 1
    return result
```
The sort+two-pointer is O(n²) time, O(1) extra space, and handles dedup naturally via sorted skip logic.

## Complexity
- **Time:** O(n²) — double loop
- **Space:** O(n) — hash map storing indices

## Code Quality
Correct but overly complex in two ways:
1. **Manual 3-element sort (lines 31-44)** — 15 lines of if/elif/else when `sorted([a, b, complement])` does the same in one line
2. **Hash map approach** — standard sort+two-pointer is simpler and O(1) space
The (0,0,0) special case is a code smell — the sort+two-pointer handles it naturally.

## Key Takeaways
- 3Sum canonical solution: sort + two-pointer (not hash map)
- Sorting enables: (a) two-pointer search, (b) natural dedup via skip
- Manual sorting of 3 elements is never needed — `sorted()` exists
- The hash map approach works but is not the interview-optimal solution
- Fixed pointer + two-pointer = standard for k-sum problems
