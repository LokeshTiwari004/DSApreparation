# Problem: 3Sum (#15)

**Difficulty:** Medium  
**Link:** https://leetcode.com/problems/3sum/  
**Date Solved:** 02/07/26  
**Phase:** Phase 3 – Mixed Set 1

---

## User's Approach

Sort + hash set + two loops:
1. Sort array, build hash set of all values
2. For each index `idx` (skipping duplicates via `num == nums[idx+1]`)
3. If `num ≠ 0`:
   - Check special case `[num, num, -2*num]` via line 18
   - Inner loop over `pt2` from `idx+1` (skipping duplicates), find `num3 = -num - num2`
   - Append if `num3 > num2 and num3 in hashset`
4. Handle `[0, 0, 0]` separately

---

## Correctness

⚠️ **Functional but fragile** — several concerns:
- **Non-idiomatic dedup** — uses `n-idx-1 and ...` as index guard (integer-as-boolean)
- **`[num, num, -2*num]` special case** — handles duplicate-value triplets but produces unsorted output like `[2, 2, -4]` instead of sorted `[-4, 2, 2]`
- **Hash set dedup is fragile** — relies on `num3 > num2` to avoid duplicates, but this misses the case `num3 == num2` (handled only through line 18 special case)
- **O(n) space** — hash set is unnecessary; canonical two-pointer is O(1) space

## Code Quality

**Strengths:**
- Correct overall logic (passes basic cases)
- Handles edge cases like `[0,0,0]`

**Issues:**
1. **Hash set instead of two-pointer** — unnecessary O(n) space
2. **Integer-as-boolean guards** — `n-idx-1 and ...` is non-idiomatic and fragile. Use `idx + 1 < n and ...`
3. **`[num, num, -2*num]` produces unsorted triplets** — output may not match expected format
4. **Complexity creep** — 24 lines for what should be 15 lines of canonical two-pointer

**Canonical Implementation:**
```python
class Solution:
    def threeSum(self, nums: list[int]) -> list[list[int]]:
        nums.sort()
        res = []
        for i in range(len(nums) - 2):
            if i > 0 and nums[i] == nums[i - 1]:
                continue  # skip duplicate first element
            lo, hi = i + 1, len(nums) - 1
            while lo < hi:
                s = nums[i] + nums[lo] + nums[hi]
                if s == 0:
                    res.append([nums[i], nums[lo], nums[hi]])
                    lo += 1
                    hi -= 1
                    while lo < hi and nums[lo] == nums[lo - 1]:
                        lo += 1  # skip duplicate second element
                    while lo < hi and nums[hi] == nums[hi + 1]:
                        hi -= 1  # skip duplicate third element
                elif s < 0:
                    lo += 1
                else:
                    hi -= 1
        return res
```

## Key Takeaways

1. **3Sum = sort + i + two pointers (lo, hi)** — never use hash set; it adds O(n) space
2. **Skip duplicates at all three positions** — compare with previous, not next
3. **Two-pointer inner loop** — `lo` moves right when sum too small, `hi` moves left when sum too large
4. **O(n²) time, O(1) space** is the canonical complexity — this is the second ever critique about 3Sum; must adopt the standard approach
5. **This is the 3rd re-attempt and still non-standard** — prioritize rewiring to canonical pattern
