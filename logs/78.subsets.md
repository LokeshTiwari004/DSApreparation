# 78. Subsets

## Problem
- **Title:** Subsets
- **Number:** #78
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/subsets/

## User's Approach
Iterative cascading. Start with `[[]]`, then for each number, extend the power set by appending the number to each existing subset.

## Correctness
Correct. This is the "cascading" approach — each new element doubles the number of subsets. Handles empty input (returns `[[]]`).

## Pattern Analysis
**Labeled: Backtracking — actually Iterative Cascading.**
The standard backtracking approach uses a decision tree:
```python
def subsets(self, nums):
    result = []
    def backtrack(start, path):
        result.append(path[:])
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()
    backtrack(0, [])
    return result
```
The iterative approach is O(2^n × n) time (building new lists) vs backtracking's O(2^n × n) as well, but the iterative version uses less recursion overhead. Both are O(2^n) in output size.

## Complexity
- **Time:** O(2^n × n) — doubling the set each iteration; each new subset copies from existing (size up to n)
- **Space:** O(2^n) — storing all subsets; user's note is correct

## Code Quality
Clean and concise (8 lines). `for set in power_set` shadows the built-in `set()` — avoid this, use `subset` instead. 10 min, no hints — well within the 20-min rule.

## Key Takeaways
- Subsets = cascading/iterative vs backtracking — know both
- The cascading approach is: start with `[[]]`, for each num → extend with `existing + [num]`
- Backtracking approach uses include/exclude decision at each index
- Total subsets = 2^n (each element has 2 choices: in/out)
- `for subset in subsets` instead of `for set in power_set` to avoid shadowing
