# 88. Merge Sorted Array

## Problem
- **Title:** Merge Sorted Array
- **Number:** #88
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/merge-sorted-array/

## User's Approach
Fill from right to left. Compare `nums1[m-1]` and `nums2[n-1]`, place the larger at the current `cursor` position. Decrement the corresponding pointer. Handle remaining elements when one array is exhausted.

## Correctness
Correct. The fill-from-right approach avoids overwriting `nums1`'s elements since the empty space is at the end. Handles all edge cases: one array empty, all elements from one array larger, etc.

## Pattern Analysis
**Pattern: Two Pointers (fill from right).**
The key insight: since `nums1` has extra space at the end, iterate backwards and place the larger element at the tail. This avoids the O(m) shift that forward-filling would require.

```python
def merge(self, nums1, m, nums2, n):
    while m and n:
        if nums1[m-1] > nums2[n-1]:
            nums1[m+n-1] = nums1[m-1]
            m -= 1
        else:
            nums1[m+n-1] = nums2[n-1]
            n -= 1
    if n:
        nums1[:n] = nums2[:n]
```
The slice-copy at the end is a common optimization.

## Complexity
- **Time:** O(m + n) — each element processed once
- **Space:** O(1) — in-place modification

## Code Quality
Clean and canonical. Uses a `for` loop with `range(m+n-1, -1, -1)` for the cursor. The truthy check `if m and n:` is Pythonic. Finally solved after being unsolved since 18/05.

## Key Takeaways
- Fill from right = standard pattern for in-place array merge
- The extra trailing space in nums1 is the key enabler
- Always check remaining elements from nums2 after the main loop
