## Pattern Analysis
**Correct pattern: Two Pointer — fill from the right.**
The blocker was trying to merge from the front, which causes overwriting conflicts. The key insight: `nums1` has `m + n` length with the last `n` slots free. Filling from the back avoids any overwriting — the largest elements go to the rightmost empty slots first.

Three pointers:
- `p1 = m - 1` (last valid element of nums1)
- `p2 = n - 1` (last element of nums2)
- `p = m + n - 1` (write position, from the back)

```python
def merge(nums1, m, nums2, n):
    p1, p2, p = m - 1, n - 1, m + n - 1
    while p2 >= 0:
        if p1 >= 0 and nums1[p1] > nums2[p2]:
            nums1[p] = nums1[p1]
            p1 -= 1
        else:
            nums1[p] = nums2[p2]
            p2 -= 1
        p -= 1
```
Note: loop only needs to run while `p2 >= 0`. If `p2` is exhausted, `nums1`'s remaining elements are already in place.

## Complexity
- **Time:** O(m + n) — each element placed exactly once
- **Space:** O(1) extra — in-place, no auxiliary array
