## Pattern Analysis
**Correct pattern: Binary Search on boundary (lower bound).**
Search Insert Position == find the first index where `nums[i] >= target`. This is the "lower bound" operation in binary search — also called `bisect_left` in Python's `bisect` module.

The user's approach uses the same custom adjacent-guard structure as #704. The standard template:

```python
def searchInsert(nums, target):
    lo, hi = 0, len(nums)  # hi = len(nums), not len(nums)-1
    while lo < hi:
        mid = lo + (hi - lo) // 2
        if nums[mid] >= target:
            hi = mid
        else:
            lo = mid + 1
    return lo
```
Using `lo < hi` with `hi = mid` is the "inclusive-exclusive" binary search pattern — runs until the search space is a single element, which is the answer. No adjacent-case special handling needed.

## Key Takeaways
- Search Insert Position == lower bound (`bisect_left`) — the first index where `nums[i] >= target`
- `hi = len(nums)` (not `len(nums) - 1`) — because the insert position could be past the last element
- Two standard binary search loops: (a) `lo <= hi` with `lo=mid+1, hi=mid-1` (search for exact match), (b) `lo < hi` with `lo=mid+1, hi=mid` (find boundary)
