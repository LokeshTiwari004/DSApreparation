## User's Final Solution
```python
def detectCycle(self, head):
    if not head:
        return head
    fast = slow = head
    flag = True
    while True:
        slow = slow.next
        fast = fast.next
        if not fast or not slow:
            return None
        if flag:
            if fast.next:
                fast = fast.next
            if fast == slow:
                if fast == head:
                    return head
                fast = head
                flag = False
        elif fast == slow:
            return fast
```

## Correctness
Correct. Phase 1 (flag=True): slow 1×, fast 2× until they meet. Phase 2 (flag=False): fast reset to head, both 1× until they meet at cycle start. Handles: null head, cycle at head (returns head directly), no cycle (returns None).

## Pattern Analysis
**Correct pattern: Floyd's Cycle Detection (Fast & Slow Pointers) + second phase.**

Standard cleaner implementation:
```python
def detectCycle(self, head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            slow = head
            while slow != fast:
                slow = slow.next
                fast = fast.next
            return slow
    return None
```

**Why Phase 2 works:** Let D = distance from head to cycle start, R = distance from cycle start to meeting point, C = cycle length. When they meet: slow = D + R, fast = D + R + kC = 2(D+R). So D = kC - R. After resetting slow to head: after D steps, slow reaches cycle start. Meanwhile, fast has traveled D from meeting point = kC - R, which lands exactly at cycle start.
