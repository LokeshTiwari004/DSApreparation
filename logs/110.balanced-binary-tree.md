# 110. Balanced Binary Tree

## Problem
- **Title:** Balanced Binary Tree
- **Number:** #110
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/balanced-binary-tree/

## User's Approach
Uses an instance variable `self.balanced` to track balance state. `height` computes subtree height recursively and sets `self.balanced = False` when `abs(right_h - left_h) >= 2`. `isBalanced` initializes the flag, calls `height`, and returns the flag.

## Correctness
Correct. The condition `-2 < right_h - left_h < 2` correctly detects when height difference exceeds 1. However, no early exit — once unbalanced is detected, height computation continues unnecessarily for the rest of the tree.

## Pattern Analysis
**Pattern: Recursive height computation with balance check.**
The instance variable approach works but the canonical pattern returns -1 from height when unbalanced:
```python
def isBalanced(self, root):
    def dfs(node):
        if not node:
            return 0
        left = dfs(node.left)
        right = dfs(node.right)
        if left == -1 or right == -1 or abs(left - right) > 1:
            return -1
        return 1 + max(left, right)
    return dfs(root) != -1
```
This eliminates the instance variable and early-exits (once -1 propagates from a subtree, all parent calls short-circuit through `left == -1 or right == -1`).

## Complexity
- **Time:** O(n) — each node visited once
- **Space:** O(height) — recursion stack; O(n) worst case
- User's "O(nodes)" and "O(height)" are correct

## Code Quality
Works but has two style issues:
1. Instance variable `self.balanced` is unnecessary — return -1 sentinel is cleaner
2. `1 + left_h if left_h > right_h else 1 + right_h` should be `1 + max(left_h, right_h)`

## Key Takeaways
- Balanced tree = every node's subtrees differ in height by at most 1
- The -1 sentinel pattern provides both early exit and avoids instance variables
- `max()` is the idiomatic way to compute max of two values
