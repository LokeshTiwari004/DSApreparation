# 206. Reverse Linked List

## Problem
- **Title:** Reverse Linked List
- **Number:** #206
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/reverse-linked-list/

## User's Approach
Iterative three-pointer reversal. `last` tracks previous node, `current` traverses. In each iteration, reassigns `current.next` to `last` before advancing both pointers. Python tuple unpacking enables single-line swap.

## Correctness
Correct. Handles: empty list (returns None), single node (loop doesn't execute, returns head itself), two+ nodes (standard reversal). The tuple unpacking is safe because Python evaluates all right-hand values before assigning.

## Pattern Analysis
**User labeled: swap var — vague.**
**Correct label: Linked List Reversal (Iterative).**

This is the single most important linked list operation. The iterative approach uses three references: `last`, `current`, and implicit `next` (saved before reassignment). The recursive approach is also standard but uses O(n) stack space.

```python
# Iterative (canonical)
def reverseList(head):
    last = None
    while head:
        nxt = head.next   # save next
        head.next = last  # reverse pointer
        last = head       # advance last
        head = nxt        # advance head
    return last
```

## Complexity
- **Time:** O(n) — each node visited once
- **Space:** O(1) — three pointer variables
- User stated O(n) / O(1) — **correct**

## Code Quality
Clean and efficient. The tuple-unpacking version is elegant but worth being able to write the explicit version too (some interviewers find the tuple swap harder to read). Pattern label needs improvement.

## Key Takeaways
- Reverse Linked List is the LL equivalent of Binary Search for arrays — memorize it cold
- Three references: `last` (prev), `current`, and save `next` before overwriting
- Python tuple unpacking is safe here (right side evaluated before assignment) but the explicit version is more universally readable
- This pattern recurs in: palindrome check (reverse 2nd half), reverse between positions, reverse in k-groups
- Recursive version exists but iterative O(1) space is preferred
