# 695. Max Area of Island

## Problem
- **Title:** Max Area of Island
- **Number:** #695
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/max-area-of-island/

## User's Approach
Standard double for-loop scanning the grid. `napo` DFS sinks islands and returns the area as `1 + sum of recursive calls`. Tracks max via `answer = max(answer, napo(r, c))`.

## Correctness
Correct. The DFS sinks cells by setting `grid[r][c] = 0` and returns the cumulative area. The double for-loop ensures every cell is checked.

## Pattern Analysis
**Pattern: Grid DFS with return value.**
Same as #200 but with area accumulation instead of counting:
```python
def maxAreaOfIsland(self, grid):
    m, n = len(grid), len(grid[0])
    def dfs(r, c):
        if 0 <= r < m and 0 <= c < n and grid[r][c]:
            grid[r][c] = 0
            return 1 + dfs(r+1, c) + dfs(r-1, c) + dfs(r, c+1) + dfs(r, c-1)
        return 0
    return max(dfs(r, c) for r in range(m) for c in range(n))
```
The return-value pattern makes this elegantly self-contained — no external counter needed.

## Complexity
- **Time:** O(M×N) — each cell visited at most once
- **Space:** O(M×N) — recursion stack worst-case

## Code Quality
Cleanest solution yet (12 lines). Uses the standard double for-loop (improvement from #200's BFS-level traversal). No extra visited set. The function name `napo` is non-standard — `dfs` or `area` would be clearer. No header notes recorded.
