# 207. Course Schedule

## Problem
- **Title:** Course Schedule
- **Number:** #207
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/course-schedule/

## User's Approach
Kahn's algorithm (BFS topological sort with source removal). Tracks "outdegree" (actual indegree — prerequisite count) and "incoming" (actual outgoing adjacency — dependents list). Starts with courses having 0 prerequisites, removes them and decrements dependents' counts. If any prerequisites remain after processing, a cycle exists.

## Correctness
Correct. The Kahn's algorithm correctly detects cycles in a directed graph. A valid topological ordering exists iff there are no cycles.

## Pattern Analysis
**Pattern: Kahn's Algorithm (BFS Topological Sort).**
Standard implementation:
```python
def canFinish(self, numCourses, prerequisites):
    indegree = [0] * numCourses
    adj = [[] for _ in range(numCourses)]
    for c, p in prerequisites:
        indegree[c] += 1
        adj[p].append(c)
    q = deque([i for i in range(numCourses) if indegree[i] == 0])
    processed = 0
    while q:
        v = q.popleft()
        processed += 1
        for neighbor in adj[v]:
            indegree[neighbor] -= 1
            if indegree[neighbor] == 0:
                q.append(neighbor)
    return processed == numCourses
```
Key improvement: counter `processed` avoids the final scan of indegree values.

## Complexity
- **Time:** O(V + E) — each vertex and edge processed once
- **Space:** O(V + E) — adjacency list + queue + indegree array
- User correctly notes worst-case O(V²) for dense graphs

## Code Quality
Correct but has misleading variable names:
- `outdegree` actually stores **indegree** (prerequisite count)
- `incoming[p]` actually stores **outgoing** edges from p (who depends on p)
Algorithm logic is sound. Studied Kahn's before solving (30 min after study). The `set()` for `incoming` values is unusual — a `list` is more efficient since we only iterate.

## Key Takeaways
- Topological sort = linear ordering where every edge u→v has u before v
- Kahn's algorithm: start with indegree-0 nodes, remove iteratively
- Cycle detection: if not all nodes processed, a cycle exists
- Variable names matter — indegree/outdegree confusion makes code harder to read
- DFS-based topological sort (post-order) is the alternative
