# 49. Group Anagrams

## Problem
- **Title:** Group Anagrams
- **Number:** #49
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/group-anagrams/

## User's Approach
For each word, builds a frequency map using a custom `construct_dict` method. Then linearly scans all existing group keys using a custom `check` function to find a matching anagram group. Appends to the matching group or creates a new one.

## Correctness
Correct. The `check` function properly compares two frequency dicts: verifies all keys/values match and total counts are equal (catches the case where `ref` has extra keys). Handles edge cases: empty string ("")  all go into one group), single-character strings.

## Pattern Analysis
**Correct pattern: HashMap with canonical anagram key.**
The standard approach: every anagram of a word maps to the same canonical key. Two canonical key choices:
1. `tuple(sorted(word))` — sort characters, use as key
2. `tuple(frequency_array)` — 26-element count array as key

Both allow O(1) hashmap lookup instead of O(G) linear scan across groups.

User's approach does O(G) scan per word (G = number of distinct groups), making the overall complexity O(n × G) — degenerate to O(n²) when all words are unique anagrams.

## Complexity
- **Time:** O(n × G × charset) — G groups, charset comparison per group — degenerates to O(n²) in worst case
- **Space:** O(n × charset) — storing all maps
- **Optimal:** O(n × L log L) with sorted key (L = average word length)
- User stated O(n × unique_anagram × charset) — **directionally correct but non-standard notation**

## Code Quality
Overly engineered. Custom `check` and `construct_dict` methods add ~20 lines to solve something that needs 4. In an interview, this signals unfamiliarity with hashmap keying patterns.

## Canonical Solution
```python
from collections import defaultdict

def groupAnagrams(strs):
    groups = defaultdict(list)
    for s in strs:
        key = tuple(sorted(s))      # canonical key: sorted characters
        groups[key].append(s)
    return list(groups.values())
```
Alternative O(n × L) key using frequency array (avoids sorting):
```python
key = tuple([0] * 26)
# ... count characters and build 26-element tuple
```

## Key Takeaways
- Canonical key pattern: map each element to a normalized form; group by that form
- `tuple(sorted(s))` is the one-line canonical key for anagram grouping — memorize it
- Linear group scan (O(G) per element) is always worse than hashmap lookup (O(1)) — avoid it
- `collections.defaultdict(list)` removes the need to check if key exists before appending
- This pattern generalizes: "group by equivalence class" = "map to canonical form, group by it"
