## User's Approach (Redo — 16/06/26)
Two-pointer from opposite ends. Left starts at index 0, right at last index. If sum < target, advance left. If sum > target, decrement right. Return 1-indexed positions when sum == target.

## Correctness
Correct. Handles all cases including the guarantee that exactly one solution exists. The while `lo < hi` condition ensures no index is reused.

## Pattern Analysis
**Pattern: Two Pointers (opposite ends).** This is the correct pattern for sorted arrays. The sorted constraint is the key hint — binary search on one element and two-pointer search on the other are both viable, but two-pointer is the simplest O(n) time / O(1) space solution.

Hash map (used in the original solve) solves the problem but at O(n) space, missing the point of the sorted variant entirely.

## Complexity
- **Time:** O(n) — single pass, each element visited at most once
- **Space:** O(1) — no extra data structures

## Code Quality
