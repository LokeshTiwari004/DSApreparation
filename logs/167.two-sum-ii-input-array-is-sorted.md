# Problem: Two Sum II — Input Array Is Sorted (#167)

**Difficulty:** Medium  
**Link:** https://leetcode.com/problems/two-sum-ii-input-array-is-sorted/  
**Date Solved:** 28/06/26  
**Phase:** Phase 3 – Mixed Set 1

---

## User's Approach

Two-pointer from opposite ends:
1. `pt1` at start (0), `pt2` at end (len-1)
2. Compute `numbers[pt1] + numbers[pt2]`
3. If sum == target: return indices + 1 (1-indexed)
4. If sum > target: `pt2 -= 1` (need smaller sum)
5. If sum < target: `pt1 += 1` (need larger sum)

---

## Correctness

✅ **Correct** — handles all cases (guaranteed exactly one solution per problem constraints):
- Target smaller than both ends → `pt2` moves left
- Target larger than both ends → `pt1` moves right
- Standard case → pointers converge to solution

---

## Pattern Analysis

**Pattern:** Two Pointers — **Correct Label**

**Why it applies:**
- Sorted array + monotonic property: increasing `pt1` increases sum, decreasing `pt2` decreases sum
- O(1) space vs hash map (O(n) space)
- This is the intended solution for the sorted variant
