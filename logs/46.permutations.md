# 46. Permutations

## Problem
- **Title:** Permutations
- **Number:** #46
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/permutations/

## User's Approach
Recursive generator using `yield`. For each index, picks the element as "first", then recursively yields permutations of the remaining elements, prepending the picked element to each.

## Correctness
Correct. The recursive decomposition is: pick one element, permute the rest, prepend. Base case: empty list yields `[]`. Handles all n! permutations.

## Pattern Analysis
**Pattern: Recursive permutation generation (backtracking via generator).**
The standard backtracking approach collects results in a list parameter:
```python
def permute(self, nums):
    result = []
    def backtrack(path, remaining):
        if not remaining:
            result.append(path[:])
            return
        for i in range(len(remaining)):
            backtrack(path + [remaining[i]], remaining[:i] + remaining[i+1:])
    backtrack([], nums)
    return result
```
The generator approach avoids the explicit `result` parameter but requires a wrapper. Both are valid. The critical insight: each recursive call reduces the problem size by one.

## Complexity
- **Time:** O(n × n!) — n! permutations, O(n) to copy each
- **Space:** O(n) — recursion stack + path (not counting output)
- User's "O(n!) time, O(n) space" is approximately correct (O(n × n!) for time due to list concatenation)

## Code Quality
Clean use of Python generators. The method name `permutations` shadows the `itertools.permutations` built-in — rename to `_permute` or similar. 30 min, no hints — slightly over the 20-min rule.

## Key Takeaways
- Permutations = pick first, permute rest (recursive problem reduction)
- n! grows fast: 5! = 120, 10! = 3,628,800 — expect large output
- Generator with `yield` is elegant Python but requires a wrapper for list output
- The recursive slicing `nums[:idx] + nums[idx+1:]` creates new lists at each level — O(n²) overhead
- Standard backtracking uses visited/set to track used elements instead of slicing
