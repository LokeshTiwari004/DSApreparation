# 130. Surrounded Regions

## Problem
- **Title:** Surrounded Regions
- **Number:** #130
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/surrounded-regions/

## User's Approach
BFS per 'O' component. Explores the entire connected component; if the BFS hits the grid boundary, returns without marking. If the BFS completes without hitting the boundary, flips all visited 'O's to 'X'.

## Correctness
Correct but **inefficient**. Boundary-connected components are re-explored from every 'O' that triggers a new `surrounded` call — the BFS returns early on boundary hit without marking visited cells, so subsequent 'O's in the same component trigger redundant BFS traversals.

## Pattern Analysis
**Pattern: Connected component labeling with boundary check.**
The standard approach avoids redundant work by reversing the logic:
1. Start DFS/BFS from all boundary 'O's, marking them as safe (e.g., 'M')
2. Flip all remaining 'O's to 'X' (they're truly surrounded)
3. Restore marked cells back to 'O'

```python
def solve(self, board):
    m, n = len(board), len(board[0])
    def dfs(r, c):
        if 0 <= r < m and 0 <= c < n and board[r][c] == 'O':
            board[r][c] = 'M'
            dfs(r+1, c); dfs(r-1, c); dfs(r, c+1); dfs(r, c-1)
    for i in range(m):
        dfs(i, 0); dfs(i, n-1)
    for j in range(n):
        dfs(0, j); dfs(m-1, j)
    for i in range(m):
        for j in range(n):
            if board[i][j] == 'O': board[i][j] = 'X'
            elif board[i][j] == 'M': board[i][j] = 'O'
```
The boundary-first approach visits each cell at most once — O(M×N).

## Complexity
- **Time:** O(M×N) per component exploration; worst-case O(M×N × C) where C is number of components that touch the boundary
- **Space:** O(M×N) — recursion/set storage
- Standard approach: O(M×N) time, O(M×N) space

## Code Quality
Has several issues:
1. **Inefficient for boundary-connected components** — re-explores from every 'O' trigger
2. **Variable shadowing** — `x, y` reassigned in inner loop, confusing
3. **Set-based BFS** (`visiting` / `visited`) is non-standard — deque is typical
4. No header notes (time/hint/pattern/complexity)
The core logic (detect boundary connectivity → flip) is correct.

## Key Takeaways
- Surrounded Regions = inverse logic: mark safe regions first
- Boundary 'O's and their connected components cannot be flipped
- Three phases: mark boundary-connected → flip interior → restore marks
- This is a variant of #200 / #695 with a boundary condition
- The boundary-first approach avoids redundant component re-exploration
