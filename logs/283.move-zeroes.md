# 283. Move Zeroes

## Problem
- **Title:** Move Zeroes
- **Number:** #283
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/move-zeroes/

## User's Approach
Two pointer in C. `zp` points to the earliest zero slot, `cursor` scans forward. When a non-zero is found and `zp` is set, swaps non-zero into `zp` position and advances `zp`. Initializes both pointers lazily (set only on first zero/non-zero encounter).

## Correctness
Correct. Handles: no zeroes (`zp` stays NULL, nothing swapped), all zeroes (no non-zero found), zeroes at end, single element. The C pointer arithmetic is correct.

## Pattern Analysis
**Correct pattern: Two Pointer (slow/fast, in-place partition).**
Classic two-pointer partition: slow pointer tracks the next available slot for non-zero elements, fast pointer scans. Equivalent mental model: "move all non-zeroes to the front, fill the rest with zeroes." This is a partition operation — same class as DNF (#75).

## Complexity
- **Time:** O(n) — single pass
- **Space:** O(1) — two pointer variables
- User stated O(n) / O(1) — **correct**

## Code Quality
Logic is correct but **written in C, not Python**. For SDE interview prep, this should be Python. C pointer dereferencing (`*zp`, `nums + i`) adds cognitive overhead that won't be present in interviews. Rewrite:

```python
def moveZeroes(nums):
    slow = 0
    for fast in range(len(nums)):
        if nums[fast] != 0:
            nums[slow], nums[fast] = nums[fast], nums[slow]
            slow += 1
```
The Pythonic version is 4 lines and clearer.

## Key Takeaways
- Two-pointer partition: slow pointer = "write position", fast pointer = "read position"
- Same structural pattern as DNF, quicksort partition, and #88 fill-from-back
- In Python: swap with tuple assignment `a, b = b, a` — no temp variable needed
- Stick to Python for all interview prep problems unless explicitly practising C/C++
