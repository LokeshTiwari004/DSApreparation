# Problem: Capacity To Ship Packages Within D Days (#1011)

**Difficulty:** Medium  
**Link:** https://leetcode.com/problems/capacity-to-ship-packages-within-d-days/  
**Date Solved:** 23/06/26  
**Phase:** Phase 3 – Pattern Track: BS on Answer

## User's Approach

Binary search on answer space `[max(weights), sum(weights)]`:
1. `lo = max(weights)`: minimum capacity = heaviest single package
2. `hi = sum(weights)`: maximum capacity = ship everything in one day
3. Binary search: for each `mid`, simulate shipping days
4. Feasibility function: iterate through weights, track remaining capacity, increment day count when weight exceeds remaining capacity
5. If `req_days > days`: capacity too small → search right half (`lo = mid + 1`)
6. Else: capacity sufficient → search left half (`hi = mid`)

---

## Correctness

✅ **Correct** — handles all cases:
- Single package → capacity = weight, returns immediately
- Packages fit exactly in D days → finds optimal capacity
- Large weights → `lo = max(weights)` ensures feasibility lower bound
- Edge case: empty weights → returns 0

## Pattern Analysis

**Pattern:** Binary Search on Answer — **Correct Label**

**Why it applies:**
- Monotonic property: if capacity `c` works, any `c' > c` also works
- Need to find minimum feasible capacity in a continuous range
- Feasibility is decidable in O(n) per check

**Canonical pattern:** Binary search on answer space with monotonic feasibility check

## Code Quality

**Strengths:**
- Clear lower/upper bound initialization
- Feasibility logic (day counting, capacity tracking) is correct and concise
- Edge case of empty weights handled explicitly

**Issues:**
1. **Non-standard binary search pattern** — `1 + (lo+hi)//2` and `(lo+hi)//2` instead of `mid + 1` and `mid`. Recomputes `mid` unnecessarily; should reuse the `mid` variable
2. **Line 29 logic redundancy** — same issue flagged in #875 and #209 critiques. Use `mid` directly
3. **No helper function** — feasibility logic is inline; extracting `can_ship(capacity)` would improve readability

**Canonical Implementation:**
```python
class Solution:
    def shipWithinDays(self, weights: List[int], days: int) -> int:
        lo, hi = max(weights), sum(weights)
        
        def can_ship(cap: int) -> bool:
            cur = cap
            d = 1
            for w in weights:
                if w > cur:
                    d += 1
                    cur = cap
                cur -= w
            return d <= days
        
        while lo < hi:
            mid = (lo + hi) // 2
            if can_ship(mid):
                hi = mid
            else:
                lo = mid + 1
        return lo
```

## Key Takeaways

1. **BS-on-answer template** — `while lo < hi`, `mid = (lo+hi)//2`, `if feasible(mid): hi = mid else: lo = mid + 1`
2. **Search bounds** — `lo = max(weights)` (must handle heaviest single item), `hi = sum(weights)` (one day fits all)
3. **Feasibility function** — extract to helper for clarity; O(n) simulation per check
4. **Monotonic property** — if capacity works, any larger capacity also works → binary search applies
5. **Same template across problems** — #875, #1011, #410 all use identical BS-on-answer structure
