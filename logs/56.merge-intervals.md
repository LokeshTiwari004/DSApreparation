# 56. Merge Intervals

## Problem
- **Title:** Merge Intervals
- **Number:** #56
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/merge-intervals/

## User's Approach
Sort by start time. Maintain answer list. For each interval, if it overlaps with the last merged interval (start <= last end), pop and merge. Otherwise, append as new entry.

## Correctness
Correct. Handles empty input, all overlapping cases, and non-overlapping intervals.

## Pattern Analysis
**Pattern: Intervals / Sorting + Greedy.** The standard approach: sort by start, then greedy merge whenever `current_start <= last_end`.

## Complexity
- **Time:** O(n log n) — dominated by sorting
- **Space:** O(n) — sorting overhead + answer list

## Code Quality
Correct but a few cleanup items:
- `from collections import deque` is imported but never used
- `answer.pop()` + `answer.append()` is wasteful — just update `answer[-1][1] = max(answer[-1][1], item[1])` in-place
- `min(last[0], item[0])` is unnecessary since sorting guarantees `item[0] >= last[0]`
- No header notes (time/pattern/complexity)

## Key Takeaways
- Overlap condition: `current_start <= last_end`
- Sort by start is the standard entry point for interval problems
- Avoid pop+push — mutating the last element is enough
- No need to re-check `min` of starts when sorted
