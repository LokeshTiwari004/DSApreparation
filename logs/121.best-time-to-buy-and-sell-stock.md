# 121. Best Time to Buy and Sell Stock

## Problem
- **Title:** Best Time to Buy and Sell Stock
- **Number:** #121
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/best-time-to-buy-and-sell-stock/

## User's Approach
Single pass tracking a running minimum price and a running maximum profit. For each price, computes profit against current minimum, updates max profit if higher, then updates minimum if current price is lower.

## Correctness
Correct. Handles all cases including monotonically decreasing arrays (returns 0), single element (returns 0), and all equal prices (returns 0).

## Pattern Analysis
**User labeled: Two Pointer — incorrect.**
**Correct pattern: Greedy.**
There is only one logical variable being tracked (the minimum seen so far). No two pointers exist. The greedy insight is: to maximize profit at day `i`, you want to have bought at the global minimum before day `i`. One pass captures this.

This is sometimes loosely called a "one-pointer" or "sliding window" but the canonical label in interview contexts is **Greedy** or **Linear Scan**.

## Complexity
- **Time:** O(n) — single pass
- **Space:** O(1) — two variables
- User stated O(n) / O(1) — **correct**

## Code Quality
Clean and readable. Variable names `profit`, `minimum` are clear. No issues.

## Key Takeaways
- "Greedy" = make the locally optimal choice at each step and it yields the global optimum
- Two Pointer requires two indices moving through the array — that's not what's happening here
- Pattern recurs in: "max profit with one transaction" style problems
- O(1) space is achievable because you only need the running min
