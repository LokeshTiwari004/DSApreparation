## User's Approach

Kahn's algorithm (BFS-based topological sort):
1. Build adjacency list for outgoing edges per vertex
2. Build indegree array (number of prerequisites per course)
3. Initialize queue with all vertices having indegree 0
4. Process queue: pop a course, add to schedule, for each neighbor decrement indegree, enqueue if it becomes 0
5. If length of schedule != numCourses, return empty array (cycle detected)

---

## Correctness

✅ **Correct** — handles all cases:
- Valid DAG → returns a valid topological ordering
- Cycle present → returns empty array
- Disconnected graph → still produces valid order
- Single course, no prerequisites → returns `[0]`

---

## Pattern Analysis

**Pattern:** Topological Sort (Kahn's Algorithm) — **Correct Label**

**Why it applies:**
- Prerequisites define a directed graph where each edge `(a, b)` means "a depends on b"
- Need to find a linear ordering respecting all dependencies
- Kahn's algorithm is the canonical BFS approach for topological sorting

## Code Quality

**Strengths:**
- Clean variable naming: `outdegree`, `incoming`, `zero_outdegree`, `course_schedule`
- Clear cycle detection: `len(course_schedule) != numCourses`
- Efficient adjacency list representation using sets for O(1) lookups

**Issues:**
1. **Variable naming is misleading** — `outdegree` key tracks how many prerequisites a course has (incoming edges, aka indegree), not outdegree. `incoming` stores outgoing edges (courses that depend on this one), not incoming edges. This is swapped from standard convention.
2. **Same naming issue as #207** — consistency with earlier learning is good, but the naming doesn't match standard graph terminology
3. **Pop from end** — `zero_outdegree.pop()` pops from the end (LIFO), producing a DFS-like order; using `pop(0)` (FIFO) would produce a BFS-like order. Both produce valid topological sorts

**Canonical Implementation:**
```python
class Solution:
    def findOrder(self, numCourses: int, prerequisites: List[List[int]]) -> List[int]:
        adj = [[] for _ in range(numCourses)]
        indegree = [0] * numCourses
        
        for course, prereq in prerequisites:
            adj[prereq].append(course)
            indegree[course] += 1
        
        q = deque([i for i in range(numCourses) if indegree[i] == 0])
        order = []
        
        while q:
            node = q.popleft()
            order.append(node)
            for neighbor in adj[node]:
                indegree[neighbor] -= 1
                if indegree[neighbor] == 0:
                    q.append(neighbor)
        
        return order if len(order) == numCourses else []
```

---

## Key Takeaways

1. **Kahn's algorithm** — standard topological sort using indegree tracking and a queue
2. **Variable naming matters** — `indegree` and `adj` are the standard conventions; avoid swapping them
3. **Cycle detection** — if processed count != total vertices, cycle exists
4. **Adjacency list vs sets** — lists are more memory-efficient for this use case (sets are overkill)
5. **Queue vs stack** — either produces a valid topological order; queue gives BFS-like order
