# 739. Daily Temperatures

## Problem
- **Title:** Daily Temperatures
- **Number:** #739
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/daily-temperatures/

## User's Approach
Monotonic decreasing stack storing indices. While loop over cursor: if `temperatures[cursor] > temperatures[stack[-1]]`, pops and computes `answer[popped] = cursor - popped`. Otherwise pushes `cursor` and advances. Stack maintains a decreasing sequence of temperatures (by index).

## Correctness
Correct. Every element is pushed once and can be popped at most once. Edge case: constant temperatures → stack fills with all indices, answer stays as default 0s. Single element → loop enters, pushes index, exits, answer stays [0].

## Pattern Analysis
**User labeled: stacks.**
**Correct label: Monotonic Stack.**
This is the defining problem for the monotonic stack pattern. The insight: for each temperature, we want the next greater element to the right. A decreasing stack maintains candidates that haven't found a warmer day yet. When a warmer day arrives, it resolves all smaller/equal temperatures on the stack.

The standard implementation uses `for` + `while` for cleaner control flow:
```python
def dailyTemperatures(self, temperatures):
    answer = [0] * len(temperatures)
    stack = []
    for i, t in enumerate(temperatures):
        while stack and t > temperatures[stack[-1]]:
            last = stack.pop()
            answer[last] = i - last
        stack.append(i)
    return answer
```

## Complexity
- **Time:** O(n) — each index pushed and popped at most once
- **Space:** O(n) — stack can hold up to n elements in worst case (decreasing sequence)
- User stated O(n) / O(n) — **correct**

## Code Quality
Clean logic, good reasoning note. The `cursor += 1` inside the `else` branch is a minor oddity — standard is `for` loop which avoids explicit cursor management. Pattern label should be "Monotonic Stack" not "stacks" — specificity matters in interviews. 48% runtime is fine; the for+while version is slightly more efficient due to CPython optimizations.

## Key Takeaways
- Monotonic stack = elements on stack are always in increasing/decreasing order
- For "next greater element" type problems → decreasing stack (resolved when a larger element arrives)
- Each element enters stack once, leaves at most once → O(n) time
- The "distance to next greater" is computed as `index_difference`, not by tracking counts
- `for` loop is cleaner than `while` with manual cursor for this pattern
