# 235. Lowest Common Ancestor of a Binary Search Tree

## Problem
- **Title:** Lowest Common Ancestor of a Binary Search Tree
- **Number:** #235
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/lowest-common-ancestor-of-a-binary-search-tree/

## User's Approach
Iterative BST traversal. Uses `min_val` and `max_val` from p, q. If root < min → go right (both nodes are right subtree). If root > max → go left (both are left subtree). Otherwise, root is the LCA.

## Correctness
Correct. BST property guarantees: if root is between p and q (or equals one), it's the LCA.

## Pattern Analysis
**Pattern: BST property traversal.**
Standard iterative version:
```python
def lowestCommonAncestor(self, root, p, q):
    while root:
        if root.val < p.val and root.val < q.val:
            root = root.right
        elif root.val > p.val and root.val > q.val:
            root = root.left
        else:
            return root
```
No need to compute min/max — direct comparison works.

## Complexity
- **Time:** O(height) — O(log n) for balanced BST, O(n) worst-case skewed
- **Space:** O(1) — iterative
- User correctly noted both

## Code Quality
Clean iterative O(1) space solution. The `p.val + q.val - min_val` trick for max is clever but `max(p.val, q.val)` is more readable. The final `elif` condition subsumes the equality cases — a bare `else` would suffice. 20 min, no hints.

## Key Takeaways
- BST LCA is simpler than general tree LCA — BST property guides direction
- Two cases: (1) both nodes on same side → go that way, (2) split or equality → return
- Iterative avoids O(height) recursion stack
- This is a direct application of the BST ordering property
- Contrast with #236 LCA of Binary Tree (no ordering helper — need DFS tracking)
