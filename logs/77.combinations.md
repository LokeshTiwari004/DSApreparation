# 77. Combinations

## Problem
- **Title:** Combinations
- **Number:** #77
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/combinations/

## User's Approach
Backtracking with include/exclude decision at each index. Include: add `start` to path, recurse with `start+1, k-1`. Exclude: skip `start`, recurse with `start+1, k`. Pruning: if remaining elements `< k`, stop.

## Correctness
Correct. The pruning optimization `stop - start + 1 < k` skips branches that can't possibly yield enough elements. The include/exclude pattern systematically explores all C(n,k) combinations without duplicates.

## Pattern Analysis
**Pattern: Backtracking (include/exclude).**
Same template as Subsets (#78) but with a depth limit `k`:
```python
def combine(self, n, k):
    result = []
    def backtrack(start, path):
        if len(path) == k:
            result.append(path[:])
            return
        for i in range(start, n + 1):
            path.append(i)
            backtrack(i + 1, path)
            path.pop()
    backtrack(1, [])
    return result
```
The `for`-loop version is more idiomatic than the include/exclude branching. Both are correct. The pruning in the user's version is equivalent to the `for` loop's natural bound.

## Complexity
- **Time:** O(C(n,k) × k) — C(n,k) = n!/(k!(n-k)!)
- **Space:** O(k) — recursion stack depth
- User's O(n^k) is a loose upper bound; C(n,k) ≤ n^k

## Code Quality
Very clean (5 min solve). Include/exclude with pruning is clear and concise. Uses `path + [start]` instead of `append`/`pop`, which creates new list copies. The `for` loop version is more standard but both are correct.

## Key Takeaways
- Combinations = Subsets with a depth limit k
- The include/exclude decision tree generates every k-sized subset exactly once
- Pruning: `remaining < k` → stop (same as loop bound `n - k + 1`)
- This is a direct variant of #78 Subsets with a stopping condition
- 5 min solve shows the backtracking pattern is internalized
