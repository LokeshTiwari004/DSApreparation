# 238. Product of Array Except Self

## Problem
- **Title:** Product of Array Except Self
- **Number:** #238
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/product-of-array-except-self/

## User's Approach
Built two auxiliary arrays: `lprod[i]` = product of `nums[0..i]` inclusive, `rprod[i]` = product of `nums[i..n-1]` inclusive. For each index `i`, result = `lprod[i-1] × rprod[i+1]`. Handled first and last elements as special cases.

## Correctness
Correct. The prefix/suffix product logic is sound. Handles first element (`rprod[1]`) and last element (`lprod[-2]`) as special cases, and middle elements via `lprod[i-1] × rprod[i+1]`. Edge case `if n < 2: return nums` correctly short-circuits (though the problem guarantees `n >= 2`).

## Pattern Analysis
**User labeled: Two Pointer — incorrect.**
**Correct pattern: Prefix + Suffix Product.**
No two pointers exist here. Two auxiliary arrays (`lprod`, `rprod`) are built independently and then combined. The correct label is "Prefix/Suffix Product" or "Left-Right Product Arrays."

The insight: `result[i] = product of all elements except nums[i]` = `product of elements to the left of i × product of elements to the right of i`.

## Complexity
- **Time:** O(n) — two passes to build arrays, one pass to compute result
- **Space:** O(n) — two auxiliary arrays of size n
- User stated O(n) / O(n) — **correct**
- **Optimal space:** O(1) extra (excluding output array) — see canonical solution

## Code Quality
Correct but verbose. Three separate index-based loops instead of two clean passes. The `range(1, n-1)` middle loop and special-casing of first/last elements add unnecessary complexity. Fix the pattern label.

## Canonical Solution
```python
def productExceptSelf(nums):
    n = len(nums)
    result = [1] * n

    # Forward pass: result[i] = product of nums[0..i-1]
    prefix = 1
    for i in range(n):
        result[i] = prefix
        prefix *= nums[i]

    # Backward pass: multiply result[i] by product of nums[i+1..n-1]
    suffix = 1
    for i in range(n - 1, -1, -1):
        result[i] *= suffix
        suffix *= nums[i]

    return result
```
This uses O(1) extra space — the output array is not counted per the problem constraints.

## Key Takeaways
- "Product except self" = left product × right product — no division needed
- Two-pass approach: forward pass builds prefix products, backward pass multiplies suffix products in-place
- O(1) extra space variant is the follow-up interviewers commonly ask — know it
- Fix: "Two Pointer" is not a valid label here — no pointer movement occurs
- This pattern (prefix + suffix scan) generalizes to: "for each index, compute something about everything to its left and right"
