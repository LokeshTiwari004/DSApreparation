# Problem: Sliding Window Maximum (#239)

**Difficulty:** Hard  
**Link:** https://leetcode.com/problems/sliding-window-maximum/  
**Date Solved:** 10/07/26  
**Phase:** Phase 3 – Pattern Track: Sliding Window

---

## User's Approach

Monotonic decreasing deque of indices:
- For each index `idx`:
  1. Pop from back while `nums[back] <= nums[idx]` (maintain decreasing order)
  2. Append current index
  3. Pop from front if it's out of the window (`idx - k + 1`)
  4. If window is full (`idx >= k-1`), append front value to result

---

## Correctness

✅ **Correct** — handles all cases:
- k = 1 → returns each element
- k = len(nums) → returns max of whole array
- Decreasing array → deque always size 1
- Increasing array → deque grows to k, front always evicted at step out

---

## Pattern Analysis

**Pattern:** Monotonic Deque — **Correct Label**

**Why it applies:**
- Need O(1) access to max of current window
- Monotonic deque maintains candidates in decreasing order
- Front = current max; back = smallest candidate for future windows
- Each element pushed/popped at most once → O(n)

## Code Quality

**Strengths:**
- Correct monotonic deque pattern
- Clean index-based front eviction check
- Good use of `collections.deque` for O(1) pop from both ends

**Issues:**
- **Minor style** — `while mono_dec:` with inner `if/else break` can be simplified to `while mono_dec and nums[mono_dec[-1]] <= num`

**Canonical pattern:**
```python
from collections import deque

class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        dq = deque()
        res = []
        for i in range(len(nums)):
            while dq and nums[dq[-1]] <= nums[i]:
                dq.pop()
            dq.append(i)
            if dq[0] < i - k + 1:
                dq.popleft()
            if i >= k - 1:
                res.append(nums[dq[0]])
        return res
```

---

## Key Takeaways

1. **Monotonic deque** — maintain decreasing order for sliding window max
2. **Indices, not values** — store indices so you can check window boundaries
3. **Three operations per element** — pop back (maintain order), push, pop front (evict out-of-window)
4. **Each element processed twice** — once added, once removed → O(n)
5. **Pattern generalizes** — min window (increasing deque), max in k-size window variations
