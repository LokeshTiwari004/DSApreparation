# 19. Remove Nth Node From End of List

## Problem
- **Title:** Remove Nth Node From End of List
- **Number:** #19
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/remove-nth-node-from-end-of-list/

## User's Approach
Two pointers with a lag counter. `cursor` advances ahead. `lagged` stays back until `lag` reaches `n`. After the loop, if `lag == n-1` → remove head. If `lag == n` → `lagged.next = lagged.next.next`.

## Correctness
Correct. The gap counter approach creates exactly an n-step gap between `cursor` and `lagged`. When `cursor` falls off the end, `lagged` is at the node just before the target. Handles: remove head (lag never reaches n), remove tail, single node.

## Pattern Analysis
**Correct pattern: Two Pointers (fixed gap).**
Standard approach: advance `fast` by n steps first, then move both `slow` and `fast` together until `fast` hits the end. `slow` ends up at the node before the target.

```python
def removeNthFromEnd(head, n):
    dummy = ListNode(0, head)
    slow = fast = dummy
    for _ in range(n):
        fast = fast.next
    while fast.next:
        slow = slow.next
        fast = fast.next
    slow.next = slow.next.next
    return dummy.next
```
The dummy node eliminates the head-removal special case entirely.

## Complexity
- **Time:** O(n) — single pass
- **Space:** O(1) — two pointer variables
- User didn't annotate — should document

## Code Quality
Correct but two issues:
1. **Lag counter instead of pre-advance** — the standard approach advances `fast` by n first, then moves both. More intuitive.
2. **No notes** — pattern, complexity, edge cases not documented
3. **Head-removal special case** (`if lag == n - 1`) — a dummy node eliminates this

## Key Takeaways
- Two-pointer fixed gap: advance fast by n, then move both → slow ends at removal point
- Dummy node removes head-removal boilerplate — use it for all LL modifications
- The lag counter approach is a valid alternative but harder to reason about
- Gap problems (nth from end, cycle detection) → two pointers
- O(1) space achievable because we work with pointers, not arrays
