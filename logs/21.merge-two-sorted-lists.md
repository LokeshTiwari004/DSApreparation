# 21. Merge Two Sorted Lists

## Problem
- **Title:** Merge Two Sorted Lists
- **Number:** #21
- **Difficulty:** Easy
- **Link:** https://leetcode.com/problems/merge-two-sorted-lists/

## User's Approach
Iterative merge. Initializes head by comparing the first nodes of both lists, then advances a `current` pointer through the merge. Uses `while True` with conditions for `list1 exists`, `list2 exists`, and `both exhausted`. Tuple-swap for advancing.

## Correctness
Correct. Handles: one empty list (base case returns the other), both empty (first `if` catches list1=None, returns list2=None), equal values (the `else` branch handles `list1.val >= list2.val`, so takes list2). All edge cases covered.

## Pattern Analysis
**Correct pattern: Linked List Merge.**
The merge is identical to merge sort's merge phase — compare heads, take the smaller, advance. A dummy node is the standard technique to avoid head-initialization boilerplate:

```python
def mergeTwoLists(list1, list2):
    dummy = ListNode()
    current = dummy
    while list1 and list2:
        if list1.val < list2.val:
            current.next = list1
            list1 = list1.next
        else:
            current.next = list2
            list2 = list2.next
        current = current.next
    current.next = list1 or list2   # one-liner for remaining tail
    return dummy.next
```
The dummy node is never used for its value — it exists just to give us a starting point for `current.next = ...` without a separate initialization block.

## Complexity
- **Time:** O(n + m) — each node visited once
- **Space:** O(1) — no new nodes, only pointer rewiring
- User stated **O(n) space — wrong.** No new nodes are allocated. Only existing pointers are reassigned. Space is O(1).

## Code Quality
Correct but two issues:
1. **Space complexity marked as O(n)** — should be O(1). The merge only rewires existing nodes, it doesn't create new ones.
2. **Head-initialization boilerplate** — 4 lines of if/elif that a dummy node eliminates entirely.
3. **Tuple swaps** — `current.next, current = list1, list1` works but is less common in LL code than `current.next = list1; current = current.next`.

## Key Takeaways
- Dummy node pattern eliminates head-initialization boilerplate — use it for all linked list construction
- `current.next = list1 or list2` handles the trailing non-exhausted list in one line
- Space complexity for pointer re-wiring is O(1) — no new nodes, no extra memory
- The merge function here is the exact same logic used in merge sort's merge step
- This pattern recurs in: merge k sorted lists, merge sort on linked lists, and merging sorted arrays
