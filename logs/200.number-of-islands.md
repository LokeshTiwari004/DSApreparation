# 200. Number of Islands

## Problem
- **Title:** Number of Islands
- **Number:** #200
- **Difficulty:** Medium
- **Link:** https://leetcode.com/problems/number-of-islands/

## User's Approach
BFS-level traversal of the grid using two deques (`current` and `next`) swapped at each diagonal level. When a `'1'` is found, calls `eatIsland` DFS to sink the entire connected component. Counts islands via increment.

## Correctness
Correct. The BFS-level traversal visits every cell. The `eatIsland` DFS correctly sinks all connected `'1'` cells. The two-queue swap mechanism traverses grid diagonals.

## Pattern Analysis
**Pattern: Grid DFS with custom traversal.**
Standard approach is a double for-loop + DFS sink:
```python
def numIslands(self, grid):
    if not grid: return 0
    count = 0
    for r in range(len(grid)):
        for c in range(len(grid[0])):
            if grid[r][c] == '1':
                count += 1
                self.dfs(grid, r, c)
    return count

def dfs(self, grid, r, c):
    if r < 0 or r >= len(grid) or c < 0 or c >= len(grid[0]) or grid[r][c] != '1':
        return
    grid[r][c] = '0'
    self.dfs(grid, r-1, c)
    self.dfs(grid, r+1, c)
    self.dfs(grid, r, c-1)
    self.dfs(grid, r, c+1)
```
The two-queue diagonal traversal is creative but unnecessary. It also shadows the built-in `next()` function.

## Complexity
- **Time:** O(M×N) — each cell visited at most once
- **Space:** O(M×N) — recursion stack for DFS (worst-case if entire grid is one island)

## Code Quality
Has several issues:
1. **Overly complex traversal** — a double for-loop is simpler and more readable
2. **`next` shadows built-in** — `next` is a Python built-in function
3. **Debug `print()` left in** — empty print on line 28
4. No notes header (time/hint/complexity/pattern)
The `eatIsland` function itself is clean and correct.

## Key Takeaways
- Number of Islands = grid traversal + sink on `'1'`
- The standard pattern is a double for-loop, not BFS/level-order traversal
- Sink uses DFS (or BFS with queue) to mark entire island as visited
- Grid DFS template: bounds check → value check → mark → recurse 4-dir
- Both #733 (flood fill) and #200 use the same sink mechanism
